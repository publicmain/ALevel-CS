📦 Demo 1: Packet Switching vs Circuit Switching

Visualize how data is split into packets and routed through the network / 可视化数据如何被拆分成数据包并通过网络路由

What is Packet Switching?
Data is broken into small packets, each with a header containing source/destination address and sequence number. Packets travel independently through the network and may take different routes. They are reassembled at the destination.

Client-Server vs P2P:
Client-Server: All messages go through the central server. Server manages, stores, and forwards messages. If server fails, no communication.
P2P: Clients communicate directly with each other. No central point of failure, but harder to manage and less secure.

💥 Demo 8: CSMA/CD - Collision Detection

Watch how Ethernet handles collisions on a shared medium / 观察以太网如何处理共享介质上的冲突

CSMA/CD = Carrier Sense Multiple Access with Collision Detection
1. Carrier Sense: Listen before sending - is the line busy?
2. Multiple Access: Multiple devices share the same medium
3. Collision Detection: If two devices send at the same time, detect the collision, stop, wait a random time, then retry
💬 Click a button to simulate CSMA/CD. Try "Both Send" to see a collision!
🎯 Exam Tip: CSMA/CD steps: (1) Listen to channel (2) If idle, transmit (3) If busy, wait (4) If collision detected, send jam signal (5) Wait random backoff time (6) Retry. Used in wired Ethernet (802.3). WiFi uses CSMA/CA (Collision Avoidance) instead.
