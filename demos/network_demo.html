📦 演示1：数据包交换 (Packet Switching)

可视化数据如何被拆分成数据包，经过不同路由到达目的地后重新组装

📖 概念讲解

生活类比：就像寄一个大包裹，快递公司把它拆成多个小包裹，每个走不同路线，到目的地再组装起来。

什么是数据包交换 (Packet Switching)？
• 数据被拆分成多个小的数据包 (Packets)
• 每个数据包都有包头 (Header)，包含：源地址、目的地址、序列号
• 数据包在网络中独立传输，可能走不同路径
• 到达目的地后，按序列号重新组装 (Reassemble)成原始数据
• 与电路交换 (Circuit Switching) 相比：更高效利用带宽、有容错能力，但可能有延迟抖动

📝 操作指南

如何操作：
1. 在下方输入框中输入一条消息（默认已有示例消息）
2. 点击「发送消息」按钮，观察消息被拆成多个彩色数据包
3. 观察每个数据包走不同路径到达目的地
4. 查看下方的数据包表格，了解每个包的状态
5. 拖动「速度」滑块可以调节动画速度

🎯 考试要点 (Exam Tips)

考试重点：
• 解释为什么数据包可能乱序到达 (arrive out of order) -- 因为走了不同路径
• 包头 (Header) 的作用：包含源/目的地址、序列号，用于路由和重组
• 数据包交换的优点：更好地利用带宽 (bandwidth)、容错性好 (fault tolerance)
• 数据包交换的缺点：有额外开销 (overhead)、实时通信可能有延迟抖动 (jitter)

🔎 演示2：DNS域名解析 (DNS Resolution)

观察域名如何一步步被解析为IP地址

📖 概念讲解

生活类比：就像查电话簿，你知道朋友的名字（域名），需要查出他的电话号码（IP地址）。

什么是DNS (Domain Name System)？
• DNS = 域名系统，相当于互联网的"电话簿"
• 把人类能读懂的域名（如 www.baidu.com）转换成机器使用的IP地址（如 110.242.68.66）
• 解析过程经过多个服务器：浏览器缓存 -> 本地DNS -> 根DNS -> 顶级域DNS -> 权威DNS
• 分为迭代查询 (Iterative) 和 递归查询 (Recursive)

📝 操作指南

如何操作：
1. 在输入框中输入一个域名（默认是 www.cambridge.org）
2. 点击「开始解析」按钮
3. 观察画布上的箭头动画，展示DNS查询的每一步
4. 查看下方日志区域的详细解析过程
💬 请输入域名并点击「开始解析」查看DNS解析过程...

🎯 考试要点 (Exam Tips)

考试重点：
• DNS层级结构：浏览器缓存 (Browser Cache) -> 本地DNS (Local DNS) -> 根DNS (Root DNS) -> 顶级域DNS (TLD DNS: .com, .org) -> 权威DNS (Authoritative DNS)
• 迭代查询 (Iterative Query) vs 递归查询 (Recursive Query) 的区别
• DNS缓存 (Cache) 的作用：加快解析速度，减少网络流量

🖧 演示3：网络拓扑结构 (Network Topologies)

交互对比星形、总线形和网状形拓扑结构

📖 概念讲解

什么是网络拓扑 (Network Topology)？
• 网络拓扑 = 网络中设备的连接方式和布局
• 星形拓扑 (Star)：所有设备连接到一个中央交换机，像星星一样辐射
• 总线形拓扑 (Bus)：所有设备共享一条主干电缆，像公交车站牌
• 网状形拓扑 (Mesh)：每台设备都与其他设备直接相连，像蜘蛛网
• 不同拓扑结构在成本、可靠性、扩展性方面各有优缺点

📝 操作指南

如何操作：
1. 点击上方的拓扑类型按钮（星形/总线形/网状形）切换查看
2. 点击画布中的任意一台电脑 (PC) 节点，可以模拟数据发送动画
3. 对比下方的拓扑对比表格
点击画布中的节点模拟数据发送

• 网状拓扑最可靠但成本最高

📚 演示4：TCP/IP 协议栈 (TCP/IP Protocol Stack)

观察数据经过每一层时如何被逐层封装

📖 概念讲解

生活类比：就像寄信 -- 信纸上写的内容=应用层数据，装进信封=传输层，写上收发地址=网络层，装进邮袋=数据链路层，邮车运送=物理层。

TCP/IP协议栈有5层：
• 应用层 (Application Layer)：产生数据（HTTP、FTP、SMTP等协议）
• 传输层 (Transport Layer)：添加端口号和序列号（TCP/UDP协议）
• 网络层 (Network Layer)：添加IP地址（IP协议，负责路由）
• 数据链路层 (Data Link Layer)：添加MAC地址和错误校验（以太网协议）
• 物理层 (Physical Layer)：转换成电信号/光信号/无线电波

📝 操作指南

如何操作：
1. 在输入框中输入一条消息（默认是 "Hello!"）
2. 点击「逐层封装」按钮
3. 观察数据从上到下经过每一层，每层添加自己的头部信息
4. 高亮的层就是当前正在处理的层
5. 应用层 (Application Layer) -- HTTP
产生HTTP请求，包含用户的消息数据
4. 传输层 (Transport Layer) -- TCP
添加源/目的端口号、序列号 | PDU名称：段 (Segment)
3. 网络层 (Network Layer) -- IP
添加源/目的IP地址 | PDU名称：包 (Packet)
2. 数据链路层 (Data Link Layer) -- Ethernet
添加MAC地址和CRC错误校验 | PDU名称：帧 (Frame)
1. 物理层 (Physical Layer)
转换为电信号/光信号/无线电波 | PDU名称：比特 (Bits)
💬 请输入消息并点击「逐层封装」查看数据封装过程...

🎯 考试要点 (Exam Tips)

考试重点：
• 每层添加自己的头部信息，这个过程叫封装 (Encapsulation)
• PDU名称要记住：数据 (Data) -> 段 (Segment) -> 包 (Packet) -> 帧 (Frame) -> 比特 (Bits)
• 接收端反向操作，逐层剥离头部，叫解封装 (De-encapsulation)
• 每层只与对端的同一层通信（逻辑上的对等通信）

💬 演示5：客户端-服务器模型 (Client-Server Model)

体验客户端如何通过中央服务器通信，对比点对点模式

📖 概念讲解

两种网络模型：
• 客户端-服务器模式 (Client-Server)：所有消息都经过中央服务器。服务器统一管理、存储和转发消息。服务器故障=所有通信中断。例：网站、电子邮件、网络游戏。
• 点对点模式 (Peer-to-Peer / P2P)：客户端之间直接通信。没有中心故障点，但管理困难、安全性较低。例：BitTorrent、区块链。

📝 操作指南

如何操作：
1. 点击上方按钮切换「客户端-服务器模式」或「点对点模式」
2. 在下方选择一个客户端（A/B/C），输入消息并发送
3. 观察消息在不同模式下的传递路径差异
4. 查看日志区域了解详细流程

🎯 考试要点 (Exam Tips)

考试重点：
• 客户端-服务器：服务器是单点故障 (Single Point of Failure)，但易于管理和安全
• P2P：去中心化 (Decentralised)，没有单点故障，但安全性低、管理困难
• 能够举出每种模式的实际应用例子
• 服务器通常比客户端更强大（更多CPU/内存/存储）

🌐 演示6：IP地址分析器 (IP Address Analyzer)

理解IPv4地址分类、公有/私有地址、二进制表示

📖 概念讲解

什么是IP地址？
• IP地址 = 网络中设备的唯一标识，就像家庭住址
• IPv4：32位，用4个十进制数表示（如 192.168.1.1），共约43亿个地址
• IPv6：128位，用8组十六进制数表示，地址空间巨大
• 公有IP (Public)：在互联网上可路由，全球唯一
• 私有IP (Private)：仅在局域网内使用，需要NAT转换才能上网

📝 操作指南

如何操作：
1. 在输入框中输入一个IPv4地址（或点击「随机IP」按钮）
2. 系统会自动分析：地址分类、是否为私有地址、二进制表示

📈 演示7：带宽计算器 (Bandwidth Calculator)

计算不同速度下传输文件所需的时间

📖 概念讲解

核心公式：传输时间 = 文件大小 / 带宽
Time = File Size / Bandwidth

注意单位换算！
• 文件大小通常用字节 (Bytes)表示，带宽通常用比特/秒 (bits per second)表示
• 1 字节 (Byte) = 8 比特 (bits)，所以要乘以8
• 十进制：1 KB = 1000 B，1 MB = 1000 KB
• 二进制：1 KiB = 1024 B，1 MiB = 1024 KiB（考试中注意区分！）

📝 操作指南

如何操作：
1. 在左侧设置文件大小（手动输入或点击快捷按钮）
2. 设置带宽速度（手动输入或点击常见速度按钮）
3. 右侧会自动计算传输时间，并显示不同速度的对比

💥 演示8：CSMA/CD 冲突检测 (Collision Detection)

观察以太网如何检测和处理共享介质上的数据冲突

📖 概念讲解

生活类比：就像在走廊里两个人迎面走来，都要让路，等一会再走。谁等的时间短，谁先走。

CSMA/CD = 载波侦听多路访问/冲突检测 (Carrier Sense Multiple Access with Collision Detection)
• 载波侦听 (Carrier Sense)：发送前先"听"一下线路是否空闲
• 多路访问 (Multiple Access)：多台设备共享同一条传输介质
• 冲突检测 (Collision Detection)：如果两台设备同时发送，检测到冲突后停止，等待随机时间后重试

完整步骤：
1. 监听信道 -> 2. 空闲则发送 -> 3. 忙则等待 -> 4. 检测到冲突则发送阻塞信号 (Jam Signal) -> 5. 随机退避等待 (Random Backoff) -> 6. 重新尝试

📝 操作指南

如何操作：
1. 点击「PC1 发送」或「PC2 发送」查看正常数据传输
2. 点击「同时发送（冲突！）」模拟两台电脑同时发送数据导致冲突
3. 观察冲突发生后的处理过程：阻塞信号 -> 随机退避 -> 重新发送
💬 点击按钮开始模拟CSMA/CD，试试「同时发送」看冲突效果！

🎯 考试要点 (Exam Tips)

考试重点：
• CSMA/CD步骤：监听(Listen) -> 空闲则发送(Transmit if idle) -> 冲突则停止(Stop on collision) -> 发阻塞信号(Jam signal) -> 随机退避(Random backoff) -> 重试(Retry)
• 用于有线以太网 (Wired Ethernet, 802.3)
• WiFi使用的是 CSMA/CA (Collision Avoidance 冲突避免)，而非CD
• 区别：CD是检测到冲突后处理，CA是提前避免冲突（先发RTS/CTS信号）
