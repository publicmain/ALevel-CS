### 二、常量与变量 (10 分钟)

#### 2.1 变量 (Variable)

【板书】变量 = 一个有名字的存储位置，里面的值可以改变

想象一个贴了标签的盒子。标签是变量名，盒子里的东西是值。你可以随时把盒子里的东西换掉。

```
DECLARE Score : INTEGER
Score ← 85         // 现在 Score 里是 85
Score ← Score + 10 // 现在 Score 里是 95
```

赋值符号是 `←`（伪代码中），Python 中用 `=`。

注意 `Score ← Score + 10` 不是数学等式！它的意思是"把 Score 的当前值加上 10，然后存回 Score"。这是初学者最容易困惑的地方。

#### 2.2 常量 (Constant)

【板书】常量 = 一旦赋值就不能改变的值

```
CONSTANT Pi = 3.14159
CONSTANT MaxStudents = 30
CONSTANT SchoolName = "Cambridge Academy"
```

【提问】既然变量已经够用了，为什么还要常量？

……两个原因：

**第一，防止意外修改。** 圆周率 π 是一个固定值，如果你不小心写了 `Pi ← 4`，编译器会报错。变量不会给你这个保护。

**第二，消灭"魔术数字 (Magic Numbers)"。** 

什么是魔术数字？看这段代码：

```
IF Count > 30
  THEN
    OUTPUT "Too many"
ENDIF
```

30 是什么？为什么是 30？三个月后你回来看这段代码，你自己都不记得了。如果你这样写：

```
CONSTANT MaxStudents = 30
...
IF Count > MaxStudents
  THEN
    OUTPUT "Too many"
ENDIF
```

一看就懂：是不是超过了最大学生数。而且如果将来改成 35，你只需要改一个地方，不需要在代码中搜索所有的"30"——万一有些 30 跟学生数没关系呢？

【板书】Magic Number 问题：
- 代码中直接出现的数字字面量
- 不知道是什么含义
- 修改时容易遗漏
- 解决方案：用有意义的常量名替代

### 三、运算符 (15 分钟)

#### 3.1 算术运算符

【板书】

| 运算符 | 含义 | 伪代码 | Python | 示例 |
|--------|------|--------|--------|------|
| 加 | Addition | + | + | 5 + 3 = 8 |
| 减 | Subtraction | - | - | 5 - 3 = 2 |
| 乘 | Multiplication | * | * | 5 * 3 = 15 |
| 除 | Division | / | / | 5 / 3 = 1.666... |
| 整除 | Integer Division | DIV | // | 5 DIV 3 = 1 |
| 取余 | Modulus | MOD | % | 5 MOD 3 = 2 |

重点说说 **MOD** 和 **DIV**，这两个是程序员的好朋友。

**DIV** 就是整数除法，只取整数部分，小数全扔掉。5 DIV 3 = 1，因为 3 只能完整地放进 5 里 1 次。

**MOD** 取余数。5 MOD 3 = 2，因为 5 除以 3 商 1 余 2。

【提问】MOD 有什么实际用途呢？

用途非常多！

1. **判断奇偶**：`n MOD 2 = 0` 就是偶数
2. **判断整除**：`n MOD 3 = 0` 就是 3 的倍数
3. **循环数组索引**：上节课队列的循环实现 `(Rear MOD MaxSize) + 1`
4. **提取数字**：`1234 MOD 10 = 4`（取个位），`1234 DIV 10 = 123`（去掉个位）
5. **时间转换**：125 分钟 = `125 DIV 60 = 2` 小时 `125 MOD 60 = 5` 分钟

#### 3.2 比较运算符

【板书】

| 运算符 | 含义 | 示例 |
|--------|------|------|
| = | 等于 | `Score = 100` |
| <> | 不等于 | `Score <> 0` |
| > | 大于 | `Score > 50` |
| < | 小于 | `Score < 0` |
| >= | 大于等于 | `Score >= 50` |
| <= | 小于等于 | `Score <= 100` |

比较运算的结果是 BOOLEAN 值：TRUE 或 FALSE。

注意伪代码中等于用 `=`，不等于用 `<>`。Python 中分别是 `==` 和 `!=`。

#### 3.3 逻辑运算符

【板书】

| 运算符 | 含义 | 示例 |
|--------|------|------|
| AND | 且 | `Age >= 18 AND Age <= 65` |
| OR | 或 | `Day = "Saturday" OR Day = "Sunday"` |
| NOT | 非 | `NOT Found` |

逻辑运算符用来组合多个条件。

一个实用的场景：判断成绩等级。
```
IF Score >= 90 AND Score <= 100
  THEN Grade ← 'A'
```

`AND` 表示两个条件必须**同时**满足。成绩必须 >= 90 并且 <= 100，才是 A。

同学们，第 9 章我们就提到过，所有程序都由三种结构组成：顺序、选择、迭代。上节课主要在用"顺序"，今天我们重点学**选择 (Selection)** 和**迭代 (Iteration)**。

### 二、选择结构 (Selection) (15 分钟)

#### 2.1 IF...THEN...ELSE...ENDIF

最基本的选择结构。

【板书】

```
IF condition
  THEN
    statements (when TRUE)
  ELSE
    statements (when FALSE)
ENDIF
```

`ELSE` 部分是可选的。如果不需要"否则"，可以不写。

嵌套 IF：

```
IF Score >= 90
  THEN
    OUTPUT "A"
  ELSE
    IF Score >= 80
      THEN
        OUTPUT "B"
      ELSE
        IF Score >= 70
          THEN
            OUTPUT "C"
          ELSE
            OUTPUT "F"
        ENDIF
    ENDIF
ENDIF
```

看到问题了吗？嵌套一深就变得很难读。这时候就该用 CASE 了。

#### 2.2 CASE...OF...ENDCASE

【板书】

```
CASE OF Grade
  "A" : OUTPUT "Excellent"
  "B" : OUTPUT "Good"
  "C" : OUTPUT "Average"
  OTHERWISE : OUTPUT "Need improvement"
ENDCASE
```

CASE 的特点：
- 适合一个变量有**多个离散值**需要判断
- 比多层嵌套 IF 清晰得多
- `OTHERWISE` 处理所有未列出的情况

【提问】什么时候用 IF，什么时候用 CASE？

……IF 适合范围判断（>= 90、>= 80 这种），CASE 适合精确匹配（等于 "A"、等于 "B" 这种）。如果你的条件是 `Score >= 50`，用 IF。如果你的条件是 `Option = 1` 或 `Option = 2` 或 `Option = 3`，用 CASE。

### 三、迭代结构 (Iteration) (22 分钟)

#### 3.1 FOR 循环

当你**知道要循环多少次**时用 FOR。

【板书】

```
FOR i ← 1 TO 10
  OUTPUT i
NEXT i
```

FOR 循环有一个控制变量 `i`，它自动从起始值增加到终止值。每次循环，i 自动加 1。

也可以用 STEP 改变步长：
```
FOR i ← 10 TO 0 STEP -2
  OUTPUT i        // 输出: 10, 8, 6, 4, 2, 0
NEXT i
```

#### 3.2 WHILE 循环

当你**不知道要循环多少次**，但知道**继续的条件**时用 WHILE。

【板书】

```
WHILE condition DO
  statements
ENDWHILE
```

特点：**先判断后执行**。如果一开始条件就是 FALSE，循环体一次都不执行。

例子：
```
Password ← ""
WHILE Password <> "secret" DO
  OUTPUT "Enter password: "
  INPUT Password
ENDWHILE
OUTPUT "Access granted!"
```

#### 3.3 REPEAT...UNTIL 循环

跟 WHILE 类似，但有两个关键区别。

【板书】

```
REPEAT
  statements
UNTIL condition
```

区别：
1. **先执行后判断**——循环体**至少执行一次**
2. 当条件为 **TRUE** 时停止（WHILE 是条件为 TRUE 时继续）

同样的密码验证用 REPEAT：
```
REPEAT
  OUTPUT "Enter password: "
  INPUT Password
UNTIL Password = "secret"
OUTPUT "Access granted!"
```

看到区别了吗？REPEAT 版本不需要给 Password 一个初始值，因为它先执行（先让用户输入），再判断。

#### 3.4 三种循环的对比

【板书】

| 特性 | FOR | WHILE | REPEAT |
|------|-----|-------|--------|
| 次数已知？ | 是 | 不一定 | 不一定 |
| 先判断/先执行 | 先判断 | 先判断 | 先执行 |
| 最少执行次数 | 0 次* | 0 次 | 1 次 |
| 停止条件 | 达到终值 | 条件为 FALSE | 条件为 TRUE |

*FOR 循环如果起始值 > 终止值，也可能 0 次。

【提问】以下三个场景分别用哪种循环？
1. 打印 1 到 100 的数字
2. 读取文件直到末尾
3. 让用户输入一个 1-10 之间的数（至少要输入一次）

……1 用 FOR（知道确切次数），2 用 WHILE（可能文件是空的），3 用 REPEAT（至少要执行一次输入）。

#### 3.5 常用循环模式

**模式一：输入验证 (Input Validation)**

确保用户输入合法数据。

【板书】

```
REPEAT
  OUTPUT "Enter a score (0-100): "
  INPUT Score
  IF Score < 0 OR Score > 100
    THEN
      OUTPUT "Invalid! Must be 0-100."
  ENDIF
UNTIL Score >= 0 AND Score <= 100
```

为什么用 REPEAT？因为你至少要让用户输入一次。

**模式二：菜单驱动程序 (Menu-Driven Program)**

【板书】

```
REPEAT
  OUTPUT "=== MENU ==="
  OUTPUT "1. Add student"
  OUTPUT "2. View students"
  OUTPUT "3. Search"
  OUTPUT "0. Exit"
  OUTPUT "Choose option: "
  INPUT Choice
  
  CASE OF Choice
    1 : CALL AddStudent()
    2 : CALL ViewStudents()
    3 : CALL SearchStudent()
    0 : OUTPUT "Goodbye!"
    OTHERWISE : OUTPUT "Invalid option!"
  ENDCASE
UNTIL Choice = 0
```

这是 REPEAT + CASE 的经典组合，很多实际程序都用这种模式。

【演示】运行 Notebook 中菜单驱动程序的示例代码

**模式三：累加器/计数器**

```
Total ← 0
Count ← 0
FOR i ← 1 TO n
  INPUT Score
  Total ← Total + Score
  IF Score >= 50
    THEN Count ← Count + 1
  ENDIF
NEXT i
Average ← Total / n
```

Total 是**累加器 (Accumulator)**，Count 是**计数器 (Counter)**。这两个模式在编程中出现的频率非常高。

### 一、为什么需要结构化编程？(5 分钟)

同学们，到目前为止我们写的代码都是从头到尾一条直线走下来的。程序短的时候没问题，但想象一下你写了一个 1000 行的程序。你需要在三个不同的地方做同样的事——比如验证用户输入。难道你要把验证的代码复制粘贴三次？

如果之后发现验证逻辑有 bug 呢？你得改三个地方。如果有一个忘了改呢？Bug 就留下来了。

【板书】DRY 原则：Don't Repeat Yourself（不要重复自己）

解决方案就是**结构化编程**：把重复的代码抽取出来，做成**过程 (Procedure)** 或**函数 (Function)**，需要的时候调用它。

---

### 二、过程与函数 (15 分钟)

#### 2.1 过程 (Procedure)

过程就是一组有名字的语句，你可以随时通过名字调用它。

【板书】

```
PROCEDURE SayHello()
  OUTPUT "====================="
  OUTPUT "  Welcome to the System"
  OUTPUT "====================="
ENDPROCEDURE

// 调用
CALL SayHello()
```

过程**不返回值**。它只是"做一件事"。

#### 2.2 函数 (Function)

函数跟过程很像，但有一个关键区别：**函数返回一个值**。

【板书】

```
FUNCTION CalculateArea(Length : REAL, Width : REAL) RETURNS REAL
  RETURN Length * Width
ENDFUNCTION

// 调用
Area ← CalculateArea(5.0, 3.0)
OUTPUT Area    // 输出 15.0
```

函数用 RETURN 返回一个值，调用时可以把返回值赋给变量。

#### 2.3 过程 vs 函数

【板书】

| | 过程 (Procedure) | 函数 (Function) |
|---|---|---|
| 返回值 | 无 | 有，且只有一个 |
| 调用方式 | `CALL ProcName()` | `Variable ← FuncName()` |
| 类比 | 一个工人去打扫房间 | 一个计算器给你答案 |

【提问】以下任务应该用过程还是函数？
1. 在屏幕上打印一条分隔线
2. 计算一个数组的平均值
3. 显示欢迎信息
4. 判断一个数是否为素数

……1 和 3 用过程（不需要返回值，只是做事），2 和 4 用函数（需要返回一个计算结果）。

### 三、参数传递 (15 分钟)

#### 3.1 什么是参数？

过程和函数可以接收**参数 (Parameter)**——告诉它要处理什么数据。

```
PROCEDURE Greet(Name : STRING)
  OUTPUT "Hello, ", Name, "!"
ENDPROCEDURE

CALL Greet("Alice")    // 输出: Hello, Alice!
CALL Greet("Bob")      // 输出: Hello, Bob!
```

`Name` 是**形式参数 (formal parameter)**——定义时的占位符。
`"Alice"` 是**实际参数 (actual parameter / argument)**——调用时传入的具体值。

#### 3.2 传值 (Pass by Value) vs 传引用 (Pass by Reference)

这是这节课最重要的概念，也是考试高频考点。

**传值 (BYVAL)**——复印机类比

【板书】想象你有一份重要文件。你的同事需要看一下。你会怎么做？

你把文件**复印一份**给他。他在复印件上涂涂画画，随便他怎么折腾——你的原件不受任何影响。

```
PROCEDURE Double(BYVAL x : INTEGER)
  x ← x * 2
  OUTPUT x    // 输出翻倍后的值
ENDPROCEDURE

Num ← 5
CALL Double(Num)
OUTPUT Num      // 仍然输出 5！原值没变！
```

传值的过程收到的是一个**副本 (copy)**。在过程内部怎么改都不影响外面的原始变量。

**传引用 (BYREF)**——原件类比

现在假设你把**原件直接交给**同事。他做的任何修改，都直接改在原件上了。

```
PROCEDURE Double(BYREF x : INTEGER)
  x ← x * 2
ENDPROCEDURE

Num ← 5
CALL Double(Num)
OUTPUT Num      // 输出 10！原值被修改了！
```

传引用传的不是副本，是变量本身。过程内部的修改**会影响外面的变量**。

#### 3.3 对比

【板书】

| | 传值 (BYVAL) | 传引用 (BYREF) |
|---|---|---|
| 传的是什么 | 值的副本 | 变量本身 |
| 原变量受影响？ | 不受影响 | 受影响 |
| 类比 | 给你一份复印件 | 把原件借给你 |
| 安全性 | 更安全（不会意外修改） | 有风险 |
| 用途 | 只需要读取数据 | 需要修改原始数据 |

【演示】运行 Notebook 中传值和传引用的对比代码

【提问】交换两个变量的值（Swap），应该用传值还是传引用？

```
PROCEDURE Swap(BYREF a : INTEGER, BYREF b : INTEGER)
  DECLARE Temp : INTEGER
  Temp ← a
  a ← b
  b ← Temp
ENDPROCEDURE
```

……必须用传引用！因为你要修改原来的两个变量。如果用传值，函数里面交换了，外面的变量纹丝不动。

**实用建议**：
- 默认用传值——更安全
- 只有当你确实需要修改原始变量时才用传引用
- 函数通常用传值 + RETURN 来返回结果

### 四、作用域 (Scope) (8 分钟)

#### 4.1 局部变量 vs 全局变量

【板书】
- **局部变量 (Local Variable)**：在过程/函数内部声明，只在内部可用
- **全局变量 (Global Variable)**：在主程序中声明，所有地方都可用

```
DECLARE Total : INTEGER    // 全局变量

PROCEDURE Calculate()
  DECLARE Temp : INTEGER   // 局部变量
  Temp ← 10
  Total ← Total + Temp
ENDPROCEDURE

CALL Calculate()
OUTPUT Total    // OK, Total 是全局的
OUTPUT Temp     // 错误! Temp 是 Calculate 的局部变量
```

#### 4.2 为什么要有作用域？

想象你在教室里写了一个名叫 `x` 的变量。隔壁班也写了一个叫 `x` 的变量。如果没有作用域的概念，这两个 `x` 就冲突了。

作用域就像教室的墙——每个过程有自己的"墙"，里面声明的变量不会跟外面冲突。

【板书】好的编程习惯：
- 尽量使用局部变量
- 全局变量越少越好
- 需要在过程之间共享数据？用参数传递，不要用全局变量

为什么？因为全局变量任何地方都能改，程序一大，你根本追踪不了是谁在什么时候改了它。这叫"副作用 (side effect)"，是 bug 的温床。

### 五、综合示例 (7 分钟)

让我们把今天学的所有内容整合成一个完整的例子：成绩管理小程序。

【演示】运行 Notebook 中综合示例

```
CONSTANT MaxStudents = 5

FUNCTION GetValidScore() RETURNS INTEGER
  DECLARE Score : INTEGER
  REPEAT
    OUTPUT "Enter score (0-100): "
    INPUT Score
  UNTIL Score >= 0 AND Score <= 100
  RETURN Score
ENDFUNCTION

FUNCTION CalculateAverage(BYVAL Scores : ARRAY, BYVAL n : INTEGER) RETURNS REAL
  DECLARE Total : INTEGER
  Total ← 0
  FOR i ← 1 TO n
    Total ← Total + Scores[i]
  NEXT i
  RETURN Total / n
ENDFUNCTION

PROCEDURE DisplayGrade(BYVAL Score : INTEGER)
  CASE OF TRUE
    Score >= 90 : OUTPUT "Grade: A"
    Score >= 80 : OUTPUT "Grade: B"
    Score >= 70 : OUTPUT "Grade: C"
    OTHERWISE   : OUTPUT "Grade: F"
  ENDCASE
ENDPROCEDURE

// 主程序
DECLARE Scores : ARRAY[1:MaxStudents] OF INTEGER
FOR i ← 1 TO MaxStudents
  Scores[i] ← GetValidScore()
  CALL DisplayGrade(Scores[i])
NEXT i
OUTPUT "Average: ", CalculateAverage(Scores, MaxStudents)
```

这个例子用到了：常量、函数（带返回值）、过程（无返回值）、参数传递（传值）、输入验证（REPEAT）、数组、FOR 循环、CASE 选择。

### 六、课堂小结 (2 分钟)

【板书】

| 概念 | 要点 |
|------|------|
| 过程 Procedure | 不返回值，CALL 调用 |
| 函数 Function | 返回值，用 RETURN |
| 传值 BYVAL | 传副本，原值不变（复印件） |
| 传引用 BYREF | 传本体，原值会变（原件） |
| 作用域 Scope | 局部优先，少用全局变量 |
| DRY 原则 | 不要重复代码，抽成过程/函数 |
