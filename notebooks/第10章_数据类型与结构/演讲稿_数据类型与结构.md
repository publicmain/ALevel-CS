### 一、为什么数据类型很重要？(8 分钟)

同学们好。今天开始第 10 章——数据类型与结构。在第 9 章我们学了怎么设计算法，现在我们要解决另一个核心问题：**算法处理的数据，到底是怎么存储和组织的？**

我先问大家一个问题。

【提问】在计算机内部，所有数据最终都是什么？

……对，都是 0 和 1，都是二进制。那问题来了：同样是 `01000001` 这 8 个比特，它到底代表什么？是数字 65？还是字母 'A'？还是布尔值 TRUE？

答案是：**数据类型告诉计算机怎么解读这些 0 和 1**。

【演示】运行 Notebook 中同一个二进制值不同解释的代码

你看，同样的 `01000001`，当我们告诉计算机"这是整数"，它就解读为 65；告诉它"这是字符"，它就解读为 'A'。数据类型不是可有可无的装饰，它决定了数据的**含义**。

数据类型还决定了三件事：

【板书】

| 作用 | 说明 |
|------|------|
| 内存分配 | BOOLEAN 只需 1 bit，REAL 需要 64 bits |
| 合法操作 | 数字可以做加减，字符串做加减就没意义 |

说到错误预防，我给大家讲一个真实的故事。1996 年，欧洲航天局发射 Ariane 5 火箭，发射后 37 秒就爆炸了。原因是什么？一个 64 位浮点数被转换成了 16 位整数，数值溢出了。就是一个**类型转换错误**，造成了 3.7 亿美元的损失。数据类型真的能决定生死。

### 二、六种基本数据类型 (20 分钟)

现在我们来逐个学习 Cambridge 考纲要求的六种基本数据类型。

#### 2.1 INTEGER (整数)

最简单的类型。没有小数部分的数字：42、-17、0 这些都是整数。

计算机用二进制补码 (Two's Complement) 存储整数。32 位整数能表示的范围是大约正负 21 亿。这在实际应用中对大部分场景够用了。

什么时候用 INTEGER？计数、索引、年龄、人数——只要是"完整的、不需要小数"的数量。

#### 2.2 REAL / FLOAT (实数/浮点数)

带小数部分的数字：3.14、-0.5、9.99。

【提问】为什么叫"浮点"数呢？"浮"的是什么？

……浮的是小数点！科学计数法 `1.2345 × 10²` 就等于 `123.45`，小数点的位置可以通过指数来"浮动"。

这里有一个特别重要的知识点，考试可能考，面试也经常问：

【演示】运行 Notebook 中 0.1 + 0.2 的代码

你们看，`0.1 + 0.2` 在计算机里不等于 `0.3`！结果是 `0.30000000000000004`。这不是 Python 的 bug，而是所有使用 IEEE 754 标准的语言都有这个问题。原因是 0.1 在二进制中是无限循环小数，就像 1/3 在十进制中是 0.333...，永远除不尽。

**教训**：永远不要用 `==` 来比较两个浮点数是否相等！

#### 2.3 CHAR (字符)

单个字符。字母 'A'、数字 '7'、符号 '@' 都是 CHAR。

底层用 ASCII 编码：每个字符对应一个数字。'A' = 65，'a' = 97，'0' = 48。

【演示】运行 Notebook 中 ASCII 码演示

有一个有趣的发现：大写字母和对应的小写字母差 32！'A' 是 65，'a' 是 97，正好差 32。这不是巧合，是 ASCII 编码故意这样设计的，方便程序做大小写转换。

注意：Python 没有单独的 CHAR 类型，字符就是长度为 1 的 STRING。但在伪代码和很多其他语言中，CHAR 是独立的类型。

#### 2.4 STRING (字符串)

由零个或多个字符组成的序列。注意"零个字符"的情况——空字符串 `""` 也是合法的字符串。

【板书】常用字符串操作（伪代码）：

| 操作 | 伪代码 | 说明 |
|------|--------|------|
| 取长度 | `LENGTH(s)` | 字符数 |
| 连接 | `s1 & s2` | 合并两个字符串 |
| 取子串 | `SUBSTRING(s, start, length)` | 截取一部分 |

字符串有一个重要特性：在大部分语言中它是**不可变的 (immutable)**。你不能修改一个字符串中的某个字符，只能创建新字符串。

#### 2.5 BOOLEAN (布尔)

只有两个值：TRUE 和 FALSE。以英国数学家 George Boole 命名。

理论上只需要 1 bit 来存储，实际上大多数系统用 1 byte，因为内存的最小寻址单位是 1 byte。

BOOLEAN 配合 AND、OR、NOT 使用，是所有条件判断的基础。

#### 2.6 DATE (日期)

表示日历中的一个日期。这里要注意不同国家格式不同：英国用 DD/MM/YYYY，美国用 MM/DD/YYYY，国际标准 ISO 8601 用 YYYY-MM-DD。

【演示】运行 Notebook 中日期格式的代码

计算机内部通常把日期存储为一个整数——从某个"纪元"开始的秒数。Unix 系统的纪元是 1970 年 1 月 1 日。

### 三、类型转换 (8 分钟)

有时候我们需要把一种类型转换成另一种，这叫**类型转换 (Type Casting)**。

【板书】

| 转换 | 函数 | 注意事项 |
|------|------|---------|
| STRING → INTEGER | `int("42")` | 字符串必须是有效数字 |
| REAL → INTEGER | `int(3.7)` → 3 | 截断，不是四舍五入！ |
| INTEGER → STRING | `str(42)` | 总是安全的 |
| CHAR → ASCII | `ord('A')` → 65 | |
| ASCII → CHAR | `chr(65)` → 'A' | |

两种类型转换方式：
- **隐式转换**：编译器自动完成，通常是安全的。比如 `5 + 3.0`，整数 5 自动变成 5.0
- **显式转换**：程序员手动指定，可能丢失信息。比如 `int(3.7)` 丢失了小数部分

【提问】`int(3.7)` 的结果是 3 还是 4？为什么？

……是 3。因为 int() 是**截断 (truncate)**，直接砍掉小数部分，不是四舍五入。要四舍五入得用 `round()`。这是考试的常考点。

### 四、记录结构 (Record) (12 分钟)

到目前为止我们学的都是**简单数据类型**，每个变量只能存一个值。但现实世界的数据通常是多个相关信息的组合。

比如一个学生，有姓名 (STRING)、年龄 (INTEGER)、成绩 (REAL)、是否在校 (BOOLEAN)。这些是不同类型的数据，但都属于同一个学生。我们需要一种方式把它们组合在一起。

这就是 **Record (记录)**——一种**复合数据类型 (composite data type)**。

【板书】伪代码定义 Record：

```
TYPE StudentRecord
    DECLARE Name : STRING
    DECLARE Age : INTEGER
    DECLARE Grade : REAL
    DECLARE IsEnrolled : BOOLEAN
ENDTYPE

DECLARE Student1 : StudentRecord
Student1.Name ← "Zhang San"
Student1.Age ← 17
Student1.Grade ← 85.5
Student1.IsEnrolled ← TRUE
```

关键语法：
- `TYPE...ENDTYPE` 定义记录的结构（相当于模板）
- 用 `.`（点号）访问字段
- 可以创建多个同类型的记录变量

【演示】运行 Notebook 中 dataclass 实现 Record 的代码

Record 的好处是什么？

1. **组织性**：相关数据放在一起
2. **可读性**：`student.name` 比 `names[i]` 更易懂
3. **可维护性**：增加新字段只需改 Record 定义

Record 是后面学面向对象编程的基础。你现在可以把 Record 想象成一张"信息卡片"的模板，每张卡上有固定的字段，但每张卡填的内容不同。

### 四、冒泡排序 (Bubble Sort) (18 分钟)

有了数组，我们自然会有一个需求：把数组里的元素**排序**。今天学最基本的排序算法——**冒泡排序 (Bubble Sort)**。

#### 4.1 基本思想

【板书】冒泡排序的核心思想：
- 从头到尾比较**相邻的两个元素**
- 如果顺序不对就交换
- 重复这个过程，直到没有交换发生

为什么叫"冒泡"？想象一杯碳酸水里的气泡，最大的气泡会最先浮到水面。冒泡排序中，每一轮都会把当前最大的元素"冒泡"到数组的末尾。

#### 4.2 逐步演示

我们用一个小数组来手动走一遍：`[5, 3, 8, 1, 2]`

【板书】第一轮：

```
[5, 3, 8, 1, 2]  比较 5 和 3 → 交换
[3, 5, 8, 1, 2]  比较 5 和 8 → 不换
[3, 5, 8, 1, 2]  比较 8 和 1 → 交换
[3, 5, 1, 8, 2]  比较 8 和 2 → 交换
[3, 5, 1, 2, 8]  第一轮结束，8 冒到了最后！
```

第二轮（最后一个已排好，只比较前 4 个）：
```
[3, 5, 1, 2, 8]  比较 3 和 5 → 不换
[3, 5, 1, 2, 8]  比较 5 和 1 → 交换
[3, 1, 5, 2, 8]  比较 5 和 2 → 交换
[3, 1, 2, 5, 8]  第二轮结束，5 到位！
```

第三轮：
```
[3, 1, 2, 5, 8]  比较 3 和 1 → 交换
[1, 3, 2, 5, 8]  比较 3 和 2 → 交换
[1, 2, 3, 5, 8]  第三轮结束，3 到位！
```

第四轮：
```
[1, 2, 3, 5, 8]  比较 1 和 2 → 不换
[1, 2, 3, 5, 8]  已排好！没有交换发生，结束。
```

【演示】运行 Notebook 中冒泡排序动画演示

#### 4.3 伪代码

【板书】

```
PROCEDURE BubbleSort(BYREF Data : ARRAY, n : INTEGER)
  DECLARE Swapped : BOOLEAN
  DECLARE Temp : INTEGER
  REPEAT
    Swapped ← FALSE
    FOR i ← 1 TO n - 1
      IF Data[i] > Data[i + 1]
        THEN
          Temp ← Data[i]
          Data[i] ← Data[i + 1]
          Data[i + 1] ← Temp
          Swapped ← TRUE
      ENDIF
    NEXT i
    n ← n - 1
  UNTIL NOT Swapped
ENDPROCEDURE
```

注意两个关键设计：
1. **Swapped 标志**：如果某一轮没有发生交换，说明已经排好了，提前结束
2. **n ← n - 1**：每轮结束后，最后一个元素已到位，不需要再比较

【演示】运行 Notebook 中伪代码对应的 Python 实现

#### 4.4 时间复杂度简介

【提问】5 个元素最多比较了几次？如果是 100 个元素呢？1000 个呢？

最坏情况下（完全倒序），5 个元素比较 4+3+2+1 = 10 次。n 个元素比较 n(n-1)/2 次。

【板书】冒泡排序的时间复杂度：O(n²)

什么意思？数据量翻倍，排序时间大约变成 4 倍。1000 个元素大约比较 50 万次，10000 个元素大约比较 5000 万次。所以冒泡排序适合小数据量，大数据量就太慢了。后面学到 A Level 内容时会学更快的排序算法。

### 五、线性搜索 (Linear Search) (10 分钟)

除了排序，另一个常见操作是**搜索**——在数组中找某个值。

#### 5.1 基本思想

线性搜索就是最简单的方法：从第一个元素开始，一个一个往后看，直到找到目标或看完所有元素。

就像你在书架上找一本书，从左到右一本一本看书名。

#### 5.2 伪代码

【板书】

```
FUNCTION LinearSearch(Data : ARRAY, n : INTEGER, Target : INTEGER) RETURNS INTEGER
  FOR i ← 1 TO n
    IF Data[i] = Target
      THEN
        RETURN i    // 找到了，返回位置
    ENDIF
  NEXT i
  RETURN -1          // 没找到
ENDFUNCTION
```

【演示】运行 Notebook 中线性搜索的代码

#### 5.3 分析

【板书】

| 情况 | 比较次数 |
|------|---------|
| 最好情况 | 1 次（第一个就是） |
| 最坏情况 | n 次（在最后一个或不存在） |
| 平均情况 | n/2 次 |

时间复杂度：O(n)——线性的，数据量翻倍，搜索时间翻倍。

线性搜索的优点是**简单**，而且**不要求数据排好序**。缺点是慢。A Level 阶段会学二分搜索 (Binary Search)，那个快得多，但要求数据已经排好序。

### 一、文件操作 (10 分钟)

之前我们学的所有数据都存在内存中——程序一关，数据就没了。如果我想把数据**永久保存**呢？这就需要**文件 (File)**。

#### 1.1 文件的基本操作

【板书】文件操作的基本步骤：打开 → 读/写 → 关闭

伪代码：
```
// 写入文件
OPENFILE "scores.txt" FOR WRITE
WRITEFILE "scores.txt", "85"
WRITEFILE "scores.txt", "92"
CLOSEFILE "scores.txt"

// 读取文件
OPENFILE "scores.txt" FOR READ
READFILE "scores.txt", Score
OUTPUT Score
CLOSEFILE "scores.txt"

// 追加到文件末尾
OPENFILE "scores.txt" FOR APPEND
WRITEFILE "scores.txt", "78"
CLOSEFILE "scores.txt"
```

三种模式：
- **READ**：只读
- **WRITE**：写入（覆盖原有内容！）
- **APPEND**：追加到末尾

还有一个常用操作：检查是否到了文件末尾。
```
WHILE NOT EOF("scores.txt")
  READFILE "scores.txt", Score
  OUTPUT Score
ENDWHILE
```

`EOF` 就是 End Of File。

【演示】运行 Notebook 中文件读写的代码

【提问】WRITE 和 APPEND 有什么区别？如果文件里已经有数据，用 WRITE 模式打开会怎样？

……对，WRITE 会**清空原有内容**重新写。APPEND 才是在后面加。这是一个很容易犯的错误——写错模式就把数据全丢了。

### 二、抽象数据类型概述 (3 分钟)

接下来我们要学三种非常重要的**抽象数据类型 (Abstract Data Type, ADT)**：栈、队列和链表。

什么叫"抽象"数据类型？意思是我们只关心它**能做什么操作**，不关心它**底层怎么实现**。就像你用自动售货机，你只知道"投币 → 选择 → 出货"，不需要知道里面的机械是怎么运作的。

实际上，这三种 ADT 都可以用数组来实现。我们今天每学一个，都会看它的概念和数组实现方式。

### 三、栈 (Stack) — LIFO (12 分钟)

#### 3.1 概念

【板书】栈 (Stack) = 后进先出 (LIFO: Last In, First Out)

最好的类比就是**一摞盘子**。你在食堂洗完盘子往上摞，最后放上去的盘子，别人拿的时候最先拿到。你不可能从底下抽一个盘子出来（除非你想摔一地盘子）。

再比如你的浏览器"后退"按钮。你依次访问了 A → B → C 三个网页，按后退，先回到 B，再按，回到 A。最后访问的页面最先被回退——这就是 LIFO。

#### 3.2 栈的操作

【板书】

| 操作 | 含义 | 类比 |
|------|------|------|
| **PUSH** | 把元素放到栈顶 | 往上摞一个盘子 |
| **POP** | 取出栈顶元素 | 拿走最上面的盘子 |
| **PEEK/TOP** | 查看栈顶元素（不取出） | 看一下最上面是什么盘子 |
| **isEmpty** | 检查栈是否为空 | 盘子摞完了吗？ |
| **isFull** | 检查栈是否满了 | 摞不下了吗？ |

#### 3.3 数组实现

用数组实现栈需要一个**栈顶指针 (Top Pointer)**。

【板书】

```
DECLARE Stack : ARRAY[1:MaxSize] OF INTEGER
DECLARE Top : INTEGER
Top ← 0   // 空栈，Top = 0

// PUSH 操作
PROCEDURE Push(Value : INTEGER)
  IF Top >= MaxSize
    THEN
      OUTPUT "Stack Overflow!"
  ELSE
    Top ← Top + 1
    Stack[Top] ← Value
  ENDIF
ENDPROCEDURE

// POP 操作
FUNCTION Pop() RETURNS INTEGER
  IF Top = 0
    THEN
      OUTPUT "Stack Underflow!"
      RETURN -1
  ELSE
    Value ← Stack[Top]
    Top ← Top - 1
    RETURN Value
  ENDIF
ENDFUNCTION
```

注意两种错误：
- **Stack Overflow**（栈溢出）：栈满了还要 PUSH
- **Stack Underflow**（栈下溢）：栈空了还要 POP

【提问】"Stack Overflow"这个词你们在哪里见过？

……对！就是那个著名的程序员问答网站的名字！它的命名就来源于这个概念。当函数递归调用太深，调用栈就会溢出。

### 四、队列 (Queue) — FIFO (12 分钟)

#### 4.1 概念

【板书】队列 (Queue) = 先进先出 (FIFO: First In, First Out)

想想你在奶茶店排队。谁先排的，谁先买到。后来的人只能排在队尾。这就是 FIFO。

计算机中到处都是队列：
- 打印任务排队
- 网络数据包排队
- CPU 任务调度排队

#### 4.2 队列的操作

【板书】

| 操作 | 含义 | 类比 |
|------|------|------|
| **ENQUEUE** | 加入队尾 | 新顾客排在队尾 |
| **DEQUEUE** | 从队首取出 | 排在最前面的顾客被服务 |
| **FRONT/PEEK** | 查看队首元素 | 看看下一个是谁 |
| **isEmpty** | 检查是否为空 | 还有人排队吗？ |
| **isFull** | 检查是否满了 | 队列排满了吗？ |

#### 4.3 数组实现（循环队列）

用数组实现队列需要两个指针：**Front (队首)** 和 **Rear (队尾)**。

但这里有一个问题。如果我们简单地实现，每次 DEQUEUE 后前面会空出位置，但新元素只能加在后面，很快数组就"用完"了——即使前面还有空位。

解决方案是**循环队列 (Circular Queue)**：当 Rear 到达数组末尾时，如果前面有空位，就绕回到数组开头。

【板书】

```
DECLARE Queue : ARRAY[1:MaxSize] OF INTEGER
DECLARE Front : INTEGER
DECLARE Rear : INTEGER
DECLARE Count : INTEGER
Front ← 1
Rear ← 0
Count ← 0

PROCEDURE Enqueue(Value : INTEGER)
  IF Count >= MaxSize
    THEN
      OUTPUT "Queue Full!"
  ELSE
    Rear ← (Rear MOD MaxSize) + 1
    Queue[Rear] ← Value
    Count ← Count + 1
  ENDIF
ENDPROCEDURE

FUNCTION Dequeue() RETURNS INTEGER
  IF Count = 0
    THEN
      OUTPUT "Queue Empty!"
      RETURN -1
  ELSE
    Value ← Queue[Front]
    Front ← (Front MOD MaxSize) + 1
    Count ← Count - 1
    RETURN Value
  ENDIF
ENDFUNCTION
```

【演示】运行 Notebook 中队列操作的演示

这里的关键是 `MOD` 运算实现"绕回"。当指针到达 MaxSize 时，`MaxSize MOD MaxSize + 1 = 1`，就回到了数组开头。

### 五、链表 (Linked List) (10 分钟)

#### 5.1 概念

数组有一个天生的限制：**大小固定**。声明时就确定了多少个元素，之后不能变。如果一开始不知道要存多少数据怎么办？

**链表 (Linked List)** 解决了这个问题。

最好的类比是**寻宝游戏**。每一条线索不仅告诉你这里藏了什么，还告诉你下一条线索在哪里。你顺着"指针"一步步找下去，直到最后一条线索说"结束了"。

【板书】链表中每个节点 (Node) 包含：
1. **数据 (Data)**：存储的值
2. **指针 (Pointer/Next)**：指向下一个节点的位置

```
[Data|Next] → [Data|Next] → [Data|Next] → NULL
```

NULL 表示链表结束，没有下一个节点了。

还需要一个**起始指针 (Start Pointer)**，告诉你链表从哪里开始。

#### 5.2 链表 vs 数组

【板书】

| 特性 | 数组 | 链表 |
|------|------|------|
| 大小 | 固定 | 可变 |
| 访问方式 | 直接用索引 O(1) | 必须从头遍历 O(n) |
| 插入/删除 | 需要移动元素 O(n) | 只需改指针 O(1) |
| 内存 | 连续分配 | 分散，每个节点额外存指针 |

#### 5.3 数组实现链表

用数组实现链表，每个元素除了存数据，还要存"下一个节点的数组下标"。

【板书】

```
TYPE Node
    DECLARE Data : INTEGER
    DECLARE Next : INTEGER    // 下一个节点的索引，-1 表示 NULL
ENDTYPE

DECLARE List : ARRAY[1:MaxSize] OF Node
DECLARE Start : INTEGER       // 链表起始位置
DECLARE Free : INTEGER        // 空闲位置的起始
```

| 索引 | Data | Next |
|------|------|------|
| 1 | 10 | 3 |
| 2 | (空) | 5 |
| 3 | 25 | 4 |
| 4 | 30 | -1 |
| 5 | (空) | -1 |

Start = 1，所以链表是：10 → 25 → 30 → NULL

Free = 2，空闲链表是：位置 2 → 位置 5

【演示】运行 Notebook 中链表操作的演示

插入和删除只需要修改 Next 指针，不用移动数据。这是链表的最大优势。
