## 第一节：为什么需要网络？（约 5 分钟）

### 1.1 想象一个没有网络的世界

【提问】如果全世界的电脑都没有联网，你想把一份文件从你的电脑发给隔壁教室的同学，怎么办？

对，你只能拿个 U 盘，拷贝好，走过去，插上他的电脑，再拷一份。这叫做"运动鞋网络"（Sneakernet）——真的，这是一个正式的术语！

如果你想把文件发给美国的朋友呢？寄个硬盘快递？那可能要一周时间。

### 1.2 网络解决了什么问题？

网络解决的核心问题其实就四个字：**资源共享**。

【板书】
```
网络的核心价值：
1. 文件共享 —— 不需要物理搬运数据
2. 硬件共享 —— 一台打印机，全办公室用
3. 通信 —— 即时消息、视频通话
4. 集中管理 —— 统一备份、统一安全策略
```

### 1.3 网络的正式定义

【板书】**网络（Network）= 两台或多台计算机通过通信介质连接起来，能够交换数据和共享资源的系统。**

注意这个定义有三个关键词：**多台计算机**、**通信介质**、**共享资源**。考试时写定义要包含这三个要素。

## 第二节：LAN vs WAN（约 10 分钟）

### 2.1 LAN：局域网

【提问】你们学校的机房里，所有电脑连在一起的那个网络，叫什么？

对，叫 **LAN——Local Area Network（局域网）**。

【板书】
```
LAN 的特征：
- 覆盖范围小：一栋楼、一个校园、一间办公室
- 所有硬件归一个组织所有（学校自己买的交换机、网线）
- 速度快：通常 100Mbps 到 10Gbps
- 成本低：因为不需要租用电信线路
```

想象一下你家里的网络：你的手机、笔记本、台式机、智能电视，都连到同一个路由器上——这就是一个最小的 LAN。

### 2.2 WAN：广域网

【提问】那如果我们学校要和北京的另一所学校建一个共享网络呢？还是 LAN 吗？

不是了，因为它跨了城市。这就是 **WAN——Wide Area Network（广域网）**。

【板书】
```
WAN 的特征：
- 覆盖范围大：城市之间、国家之间、甚至全球
- 需要租用第三方线路（电信运营商的光纤、卫星链路等）
- 速度相对较慢（因为距离远）
- 成本高：需要付费给运营商
```

**最大的 WAN 是什么？** 就是 Internet——互联网。它是无数个 LAN 通过 WAN 连接起来的巨型网络。

### 2.3 对比总结

【演示】打开 Notebook 01，运行 LAN vs WAN 的概念图。

大家看这张图。左边是一个 LAN——一栋楼里面，几台电脑通过一个交换机连在一起。右边是另一个城市的 LAN。中间通过路由器和广域网链路连接——这就形成了一个 WAN。

【板书】

| 特征 | LAN | WAN |
|:---|:---|:---|
| 范围 | 小（一栋楼） | 大（跨城市/国家） |
| 硬件归属 | 自己拥有 | 租用运营商线路 |
| 速度 | 快 | 相对慢 |
| 成本 | 低 | 高 |

## 第三节：Client-Server vs Peer-to-Peer（约 10 分钟）

现在我们知道了网络的类型，接下来要问一个更深层的问题：**网络中的电脑之间是什么关系？**

有两种基本模型。

### 3.1 Client-Server 模型

【提问】你们打开浏览器访问 B 站的时候，你的电脑和 B 站的服务器是什么关系？

你的电脑是**客户端（Client）**——它发出请求："我要看这个视频。" B 站的服务器收到请求后，把视频数据发给你。这就是 **Client-Server 模型**。

【板书】
```
Client-Server 模型：
- Server（服务器）：提供服务、集中存储数据、管理权限
- Client（客户端）：发送请求、接收服务
- 关系：不对等——Server 是"老板"，Client 是"员工"
```

优点：
- 集中管理：管理员只需要管好服务器，所有数据都在那里
- 安全性高：权限、备份、杀毒都在服务器端统一处理
- 容易扩展：加一台客户端很简单

缺点：
- 服务器挂了，所有人都受影响（单点故障）
- 服务器很贵——需要高性能硬件
- 需要专业的网络管理员

### 3.2 Peer-to-Peer（P2P）模型

【提问】有人用过迅雷下载或者种子下载吗？那个下载方式和从网站直接下载有什么不同？

种子下载的时候，你不是从一个服务器下载的——你同时从好几个人那里各下载一小块，你自己下载好的部分也会上传给别人。**每个人既是客户端，也是服务器。** 这就是 P2P。

【板书】
```
Peer-to-Peer 模型：
- 每台电脑（Peer）地位平等
- 每台电脑既是客户端，又是服务器
- 没有集中管理
- 适合小规模、临时的文件共享
```

优点：
- 不需要昂贵的服务器
- 没有单点故障——一台挂了其他的照样工作
- 搭建简单

缺点：
- 管理困难——没有集中控制
- 安全性差——每台电脑都可能有安全漏洞
- 不适合大规模网络

【演示】运行 Notebook 01 中的 Client-Server vs P2P 可视化代码。

大家看这两张图的区别。Client-Server 像一个星形——所有客户端都指向中间的服务器。P2P 像一个全互联的网格——每个人都和其他所有人连接。

## 第四节：Thin Client vs Thick Client（约 5 分钟）

在 Client-Server 模型中，还有一个有趣的问题：**客户端到底有多"厚"？**

### 4.1 Thin Client（瘦客户端）

想象你去网吧，发现那些电脑配置很低——可能连硬盘都没有。你所有的操作其实都在服务器上完成，你面前的电脑只是负责显示画面和接收你的键盘鼠标输入。这就是 **Thin Client**。

【板书】
```
Thin Client（瘦客户端）：
- 处理能力弱，依赖服务器做大部分计算
- 优点：便宜、易管理、安全（数据不在本地）
- 缺点：需要强大的服务器和稳定的网络连接
- 例子：Chrome OS 设备、虚拟桌面
```

### 4.2 Thick Client（胖客户端）

你自己的笔记本电脑就是一个 Thick Client——它有自己的 CPU、硬盘、操作系统，大部分计算都在本地完成，只偶尔需要连服务器。

【板书】
```
Thick Client（胖客户端）：
- 有自己的处理能力，能独立运行软件
- 优点：不完全依赖网络，断网也能工作
- 缺点：贵、需要单独维护和更新
- 例子：普通的笔记本电脑、台式机
```

【演示】运行 Notebook 01 中的 Thin vs Thick Client 可视化。

大家看这张图。Thin Client 那边，服务器画得很大（因为它干了大部分活），客户端画得很小。Thick Client 那边反过来——客户端自己就很强大，服务器只是起辅助作用。

## 第五节：网络拓扑（约 12 分钟）

这是今天最重要也最有趣的部分。

### 什么是拓扑？

**拓扑（Topology）** 就是网络中设备的**物理连接方式**——谁和谁连在一起，怎么连的。

你可以把它想象成城市的道路规划。同样是连接 10 个小区，你可以把它们串成一条直线，也可以围成一个环，也可以全部通向一个中心广场。不同的布局有不同的优缺点。

### 5.1 Bus Topology（总线拓扑）

【板书】
```
Bus 拓扑：
- 所有设备连接到一根主缆线（backbone）
- 数据沿着主线广播，所有设备都能"听到"
- 两端需要终结器（terminator）防止信号反射
```

【演示】运行 Notebook 01 中的 Bus Topology 可视化。

大家看，这就像一条公交线路——所有站点（设备）都在同一条路上。数据就像一辆广播车，沿着这条路开过去，所有人都能听到，但只有目标地址匹配的设备才会接收。

优点：简单、便宜、用线少。

缺点很明显：
- 主缆线断了，整个网络瘫痪
- 设备越多，碰撞越多，速度越慢
- 排查故障很难——到底是哪段线坏了？

### 5.2 Star Topology（星形拓扑）

【板书】
```
Star 拓扑：
- 所有设备连接到一个中心设备（通常是交换机 Switch）
- 数据通过中心设备转发
```

【演示】运行 Notebook 01 中的 Star Topology 可视化。

这是今天最常见的拓扑。你家里的网络就是星形拓扑——所有设备都连到路由器/交换机上。

优点：
- 一台设备的线坏了，不影响其他设备
- 容易增加新设备
- 排查故障简单——哪个端口没反应就是哪个出了问题

缺点：
- 中心设备（交换机）坏了，全网瘫痪
- 用线比 Bus 多——每台设备都要单独一根线到中心

【提问】为什么现实中 Star 拓扑比 Bus 拓扑用得多得多？

因为它的缺点很容易用冗余来解决——你可以准备一台备用交换机，但 Bus 拓扑的主缆线断了就真的没办法了。

### 5.3 Mesh Topology（网状拓扑）

【板书】
```
Mesh 拓扑：
- Full Mesh（全网状）：每台设备都和所有其他设备直接相连
- Partial Mesh（部分网状）：只有部分设备之间有直接连接
- 连接数公式（Full Mesh）：n(n-1)/2
```

【演示】运行 Notebook 01 中的 Mesh 可视化和连接数计算代码。

【提问】如果有 10 台设备做 Full Mesh，需要多少条连接？

用公式算：10 × 9 / 2 = 45 条！这就是 Mesh 的问题——设备一多，线缆数量爆炸式增长。

优点：
- 极高的可靠性——一条线断了，数据可以走别的路
- 互联网的骨干网就是 Mesh 结构

缺点：
- 非常昂贵——线缆和端口数量巨大
- 管理复杂

### 5.4 Hybrid Topology（混合拓扑）

现实中的大型网络几乎都是混合拓扑。比如学校里每个教室内部是 Star 拓扑，教室之间通过一根主干线连接——这就是 Star-Bus 混合拓扑。

【演示】运行 Notebook 01 中的 Hybrid Topology 可视化。

### 5.5 拓扑对比总结

【板书】

| 拓扑 | 可靠性 | 成本 | 扩展性 | 适用场景 |
|:---|:---|:---|:---|:---|
| Bus | 低 | 最低 | 差 | 临时小网络 |
| Star | 中等 | 中等 | 好 | 办公室、家庭 |
| Mesh | 最高 | 最高 | 差 | 骨干网、军事 |
| Hybrid | 可调节 | 可调节 | 好 | 大型企业 |

## 第一节：网络接口卡 NIC（约 5 分钟）

### 1.1 NIC 是什么？

你的电脑能上网，最基本的一个前提是什么？它得有一个**网络接口卡（NIC — Network Interface Card）**。

【板书】
```
NIC (Network Interface Card)：
- 把计算机连接到网络的硬件
- 每个 NIC 有一个全球唯一的 MAC 地址
  （MAC = Media Access Control）
- MAC 地址格式：XX:XX:XX:XX:XX:XX（6 组十六进制）
```

### 1.2 NIC 做了什么？

NIC 的工作其实分为两个方向：

**发送时**：把计算机内部的数据打包成网络能传输的帧（frame），加上源 MAC 地址和目标 MAC 地址，然后发送到网络上。

**接收时**：监听网络上的数据，检查目标 MAC 地址是不是自己的。如果是，就接收；如果不是，就忽略。

### 1.3 有线 vs 无线 NIC

【提问】你们的笔记本电脑同时支持有线和无线上网，说明它有几个 NIC？

答案是**两个**——一个有线 NIC（RJ-45 接口），一个无线 NIC（WNIC，通常集成在主板上）。它们有**不同的 MAC 地址**。

【演示】运行 Notebook 02 中的 MAC 地址格式展示代码。

注意 MAC 地址的格式：6 组十六进制数，用冒号或连字符分隔。比如 `AA:BB:CC:DD:EE:FF`。前三组通常标识制造商（比如苹果的设备前三组是固定的），后三组是序列号。这就确保了全球每个 NIC 的 MAC 地址都是唯一的。

## 第二节：交换机 vs 集线器（约 8 分钟）

### 2.1 Hub（集线器）——"笨"设备

Hub 是早期网络设备。它的工作原理极其简单：**收到一个数据包，无脑转发给所有端口**。

【提问】这有什么问题？

想象一下：你在一个 10 人的房间里，你想跟小明说句悄悄话。但 Hub 的做法是——你说的话被广播给了所有 10 个人。这有两个问题：
1. **隐私问题**——所有人都能听到
2. **效率问题**——如果两个人同时说话，就会"碰撞"（collision），谁的信息都传不过去

### 2.2 Switch（交换机）——"聪明"设备

Switch 做了一件 Hub 做不到的事：**它维护一张 MAC 地址表**。

【板书】
```
Switch 的 MAC 地址表：
端口 1 → AA:BB:CC:11:22:33
端口 2 → AA:BB:CC:44:55:66
端口 3 → AA:BB:CC:77:88:99
...
```

Switch 会学习——它记住每个端口连接的设备的 MAC 地址。当收到一个数据包时，它只把数据发送到**目标 MAC 地址所在的端口**，而不是所有端口。

【演示】运行 Notebook 02 中的 Hub vs Switch 转发行为模拟。

大家看这两张图的区别：
- Hub：数据从 A 发出，所有人都收到了（红色箭头到处都是）
- Switch：数据从 A 发给 C，只有 C 收到（只有一条绿色箭头）

这就是为什么现在几乎没有人用 Hub 了——Switch 在各方面都更优。

【板书】

| 特征 | Hub | Switch |
|:---|:---|:---|
| 转发方式 | 广播到所有端口 | 只发到目标端口 |
| MAC 地址表 | 没有 | 有，会自动学习 |
| 碰撞 | 多（整个网络是一个碰撞域） | 少（每个端口独立） |
| 安全性 | 差 | 好 |
| 价格 | 便宜 | 稍贵 |

## 第三节：路由器（约 8 分钟）

### 3.1 Router 做什么？

【提问】Switch 已经很聪明了，为什么还需要 Router？

答案是：**Switch 只能在同一个网络（LAN）内部转发数据，而 Router 能在不同网络之间转发数据。**

打个比方：Switch 像是一栋楼的门卫——它认识楼里每个人，能帮你把快递送到正确的房间号。但如果你要寄一个包裹到另一个城市呢？你需要一个快递站——这就是 Router。

【板书】
```
Router（路由器）：
- 连接不同的网络（LAN 与 LAN，LAN 与 WAN）
- 工作在 IP 地址层面（而 Switch 工作在 MAC 地址层面）
- 维护路由表（routing table），记录到达各个网络的路径
- 能选择最佳路径转发数据
```

### 3.2 Router vs Switch — 关键区别

【板书】

| 特征 | Switch | Router |
|:---|:---|:---|
| 工作层级 | MAC 地址 | IP 地址 |
| 连接范围 | 同一 LAN 内 | 不同网络之间 |
| 转发依据 | MAC 地址表 | 路由表 |
| 广播处理 | 转发广播 | 隔离广播 |

### 3.3 Router 的工作原理

Router 收到一个数据包后，会做这几件事：
1. 查看数据包的**目标 IP 地址**
2. 查询**路由表**——这个 IP 地址属于哪个网络？该往哪个方向转发？
3. 把数据包转发到**最佳路径**上的下一个路由器
4. 重复这个过程，直到数据到达目标网络

大家看这张图：Network A 和 Network B 是两个独立的 LAN，各自有自己的 Switch。它们通过一个 Router 连接起来。Router 上面还连着互联网（云的图标）。所以 Router 是三个网络的交汇点。

### 4.1 WAP（Wireless Access Point）

你家里的 WiFi 路由器其实是三个设备的合体：Router + Switch + WAP。但在企业环境中，这三个通常是分开的。

## 第五节：传输介质（约 12 分钟）

好，硬件设备我们了解了。现在来看**数据到底是通过什么"路"传输的**。

### 5.1 铜缆

#### 双绞线（Twisted Pair Cable）

这是最常见的网络线缆，你们平时见到的那种水晶头网线就是双绞线。

【提问】为什么要把铜线扭在一起，而不是平行铺设？

因为**扭在一起可以减少电磁干扰**。平行的两根线在电磁场中容易产生互相干扰（串扰），扭在一起后，干扰信号在两根线上的影响可以相互抵消。

【板书】
```
双绞线（Twisted Pair）：
- UTP（非屏蔽双绞线）：便宜，常用于办公室
- STP（屏蔽双绞线）：多一层金属屏蔽，抗干扰更好
- 速度：Cat5e 可达 1Gbps，Cat6 可达 10Gbps
- 距离限制：约 100 米
```

#### 同轴电缆（Coaxial Cable）

这个你们可能在电视天线线上见过。它的结构是：中间一根铜芯，外面包一层绝缘体，再包一层金属网（屏蔽层），最外面是塑料外皮。

比双绞线抗干扰能力强，但比较粗硬，不方便布线。早期网络用得多，现在主要用于有线电视和某些广域网连接。

### 5.2 光纤（Fibre Optic Cable）

这是目前最先进的有线传输介质。

【提问】光纤传输数据用的是什么？电信号？

不是！光纤用的是**光信号**。数据被转化为光脉冲——有光代表 1，没光代表 0——在一根玻璃纤维中传输。

【板书】
```
光纤（Fibre Optic）：
- 传输介质：光信号通过玻璃/塑料纤维
- 原理：全内反射（Total Internal Reflection）
- 优点：
  ✓ 极快（可达数百 Gbps）
  ✓ 传输距离远（数十公里不需要中继）
  ✓ 不受电磁干扰
  ✓ 难以窃听（安全性高）
- 缺点：
  ✗ 昂贵
  ✗ 安装和维修需要专业设备
  ✗ 弯折容易断裂
```

【演示】运行 Notebook 02 中的光纤全内反射原理图和速度对比图。

大家看这个光纤横截面：最中间是纤芯（core），光在里面传播。外面是包层（cladding），它的折射率比纤芯低，所以光在纤芯和包层的界面会发生**全内反射**——光束在纤芯里不断反弹前进，就像在一根管子里弹来弹去，不会跑出去。

### 5.3 无线传输

#### WiFi

WiFi 使用**无线电波（Radio Waves）** 在 2.4GHz 或 5GHz 频段传输数据。

【板书】
```
WiFi：
- 频段：2.4GHz（穿墙强，但慢、干扰多）
       5GHz（快，但穿墙弱）
- 范围：通常 30-50 米（室内）
- 速度：WiFi 6 可达数 Gbps
- 安全问题：信号在空气中传播，可能被截获
```

#### 微波（Microwave）

微波通信用于点对点的长距离传输，比如两栋楼之间、或城市之间的基站连接。

关键限制：**需要视线传播（Line of Sight）**——发射器和接收器之间不能有障碍物。

#### 卫星通信（Satellite）

卫星通信把信号从地面站发射到太空中的卫星，卫星再转发到另一个地面站。

优点：覆盖范围极广，连海上和偏远地区都能覆盖。
缺点：**延迟高**——信号要飞到 36,000 公里高的地球同步轨道再飞回来，即使是光速也需要约 0.5 秒的往返时间。

| 特征 | 有线 | 无线 |
|:---|:---|:---|
| 速度 | 更快 | 较慢 |

| 移动性 | 差（被线缆限制） | 好 |

### 数据传输速率

【板书】
```
关键公式：
传输时间 = 文件大小 / 带宽
注意单位统一：文件通常用 Bytes，带宽通常用 bits/s
1 Byte = 8 bits

换算：
1 Kbps = 1,000 bps
1 Mbps = 1,000,000 bps
1 Gbps = 1,000,000,000 bps
```

【提问】一个 800MB 的文件，用 100Mbps 的网络下载，需要多少时间？

## 第一节：Internet vs WWW（约 8 分钟）

### 1.1 Internet（互联网）

**Internet = 全球最大的网络基础设施。** 它是由无数个 LAN 和 WAN 通过路由器互联而成的一个全球性网络。

注意，Internet 是"路"——是光纤、铜缆、卫星链路、路由器这些物理设施构成的**基础设施**。

【板书】
```
Internet = 基础设施（Infrastructure）
- 本质：全球互联的网络的网络（a network of networks）
- 使用 TCP/IP 协议
- 起源：1969 年的 ARPANET
```

### 1.2 WWW（万维网）

**WWW = 运行在 Internet 上的一种服务。** 它是一个由超文本文档（网页）通过超链接（hyperlinks）互相连接形成的信息系统。

打个比方：如果 Internet 是公路网，那 WWW 就是公路上跑的**快递系统**——它利用公路来运送包裹（网页），但公路本身不是快递系统。

【板书】
```
WWW = 运行在 Internet 上的服务（Service）
- 本质：超文本文档系统
- 使用 HTTP/HTTPS 协议
- 由 Tim Berners-Lee 在 1989 年发明
- 通过浏览器访问
```

### 1.3 Internet 上不只有 WWW！

这是很多人搞混的原因——我们平时说"上网"大部分时间确实在用 WWW（浏览网页）。但 Internet 上还运行着很多其他服务：

【板书】
```
Internet 上的服务（WWW 只是其中之一）：
- WWW（网页浏览）—— 用 HTTP/HTTPS
- Email（电子邮件）—— 用 SMTP/POP3/IMAP
- FTP（文件传输）—— 用 FTP 协议
- VoIP（网络电话）—— 用 SIP 协议
- DNS（域名解析）—— 用 DNS 协议
- 在线游戏、视频流、即时通讯……
```

## 第二节：IP 地址（约 10 分钟）

### 2.1 什么是 IP 地址？

网络中的每一台设备要通信，都需要一个**身份标识**。就像每个家庭都需要一个门牌号才能收到快递一样。这个"门牌号"就是 **IP 地址（Internet Protocol Address）**。

### 2.2 IPv4

【板书】
```
IPv4 地址：
- 格式：4 组十进制数，用点分隔
  例如：192.168.1.100
- 每组范围：0 ~ 255（因为每组用 8 bit 存储）
- 总共 32 bit（4 × 8 = 32）
- 最多可寻址：2^32 = 约 43 亿个地址
```

【演示】运行 Notebook 03 中的 IP 地址二进制表示代码。

192.168.1.100 的二进制是什么？

192 = 11000000
168 = 10101000
1   = 00000001
100 = 01100100

所以完整的二进制表示是：11000000.10101000.00000001.01100100

### 2.3 IPv6

【提问】43 亿个地址，听起来很多？地球上有多少人？80 亿。一个人通常有几个联网设备？手机、电脑、平板、手表……IPv4 的地址够用吗？

完全不够！这就是为什么我们需要 **IPv6**。

【板书】
```
IPv6 地址：
- 格式：8 组十六进制数，用冒号分隔
  例如：2001:0db8:85a3:0000:0000:8a2e:0370:7334
- 总共 128 bit
- 最多可寻址：2^128 ≈ 3.4 × 10^38 个地址
  （给地球上每一粒沙子分一个 IP 地址都还有富余）
```

### 2.4 公有 IP vs 私有 IP

【板书】
```
公有 IP (Public IP)：
- 全球唯一，可在互联网上路由
- 由 ISP（运营商）分配

私有 IP (Private IP)：
- 只在局域网内使用，不能直接在互联网上路由
- 常见范围：192.168.x.x、10.x.x.x、172.16-31.x.x
- 通过 NAT（网络地址转换）访问外网
```

你家里路由器分配给你手机的 192.168.1.x 就是私有 IP。你的邻居家里也可能有一个 192.168.1.x——不会冲突，因为它们在不同的 LAN 里。但你家路由器对外（面向运营商）有一个公有 IP——这个是全球唯一的。

## 第三节：URL 结构（约 5 分钟）

### 什么是 URL？

**URL = Uniform Resource Locator（统一资源定位符）**——就是网页的"地址"。

【板书】拆解一个 URL：
```
https://www.example.com:443/products/search?q=phone&page=2#results
  │        │          │  │     │              │            │
  协议   子域名     域名 端口   路径          查询参数      锚点
```

- **协议（Protocol）**：https——使用加密的 HTTP

- **查询参数（Query String）**：?q=phone&page=2——搜索条件
- **锚点（Fragment）**：#results——页面内的定位

## 第四节：DNS（约 10 分钟）

### 4.1 为什么需要 DNS？

你每天访问 baidu.com、bilibili.com，但计算机之间通信用的是 IP 地址啊！计算机不认识"baidu.com"——它只认识 220.181.38.148 这样的数字。

那谁来帮你把 "baidu.com" 翻译成 "220.181.38.148"？

答案就是 **DNS——Domain Name System（域名系统）**。DNS 就是互联网的"电话簿"。

### 4.2 DNS 解析过程

【板书】
```
你在浏览器输入 www.example.com，发生了什么？

1. 浏览器检查本地缓存——之前访问过吗？
2. 没有 → 问操作系统缓存
3. 没有 → 问 Recursive DNS Server（通常是运营商的 DNS）
4. 没有 → 问 Root DNS Server（全球 13 组）
   → Root 说："我不知道具体 IP，但 .com 的事你去问 .com 的 TLD 服务器"
5. 问 .com TLD Server
   → 说："example.com 的权威 DNS 是 ns1.example.com"
6. 问 Authoritative DNS Server
   → 回答："www.example.com 的 IP 是 93.184.216.34"
7. 递归 DNS 把结果缓存起来，返回给浏览器
8. 浏览器用 93.184.216.34 连接目标服务器
```

这个过程听起来很复杂，但实际上通常在**几十毫秒内完成**，因为大量结果被缓存了。

## 第六节：Ethernet 和 CSMA/CD（约 8 分钟）

### 6.1 什么是 Ethernet？

**以太网（Ethernet）** 是目前最广泛使用的 LAN 技术标准。它定义了数据在网络中如何封装成帧（frame）、如何寻址（用 MAC 地址）、以及如何处理多台设备同时发送数据的冲突。

### 6.2 CSMA/CD

全称：**Carrier Sense Multiple Access with Collision Detection**

这名字很长，但其实逻辑很简单。我们用"在走廊里说话"来类比：

【板书】
```
CSMA/CD 工作流程：
1. Carrier Sense（载波侦听）：
   先听——走廊里有人在说话吗？
2. 如果没人说话 → 开始发送数据
3. 如果在发送过程中检测到碰撞（两个人同时开口了！）：
   a. 立即停止发送
   b. 发送一个 Jam Signal（告诉所有人"出事了！"）
   c. 等待一个随机时间（Random Backoff）
   d. 重新从步骤1开始
```

【提问】为什么等待时间要是"随机的"而不是固定的？

因为如果两个人都等相同的时间，它们又会同时开始说话，又碰撞！随机等待时间让它们错开，减少再次碰撞的概率。

【演示】运行 Notebook 03 中的 CSMA/CD 碰撞检测模拟。

注意：CSMA/CD 主要用于**早期的半双工以太网**。现在的全双工交换式以太网基本不会发生碰撞，因为 Switch 为每个连接提供了独立的通道。但考试仍然会考这个概念。

### 7.1 什么是云计算？

【提问】你们有没有用过 Google Docs？你的文件存在哪里？

不是你的电脑里——是在 Google 的服务器上。你通过网络访问和编辑它。这就是**云计算**的典型应用。

```
云计算（Cloud Computing）：
```

- **私有云**：一个组织独享自己的云基础设施（安全但贵）

## 第八节：Bit Streaming（约 5 分钟）

### 什么是 Bit Streaming？

Bit Streaming 就是数据的**流式传输**——数据一边下载一边播放，不需要等全部下载完。

【提问】你在 B 站看视频的时候，需要等整个视频下载完才能开始看吗？

不需要！你点了播放按钮，几秒钟后就开始播放了，视频在后台继续下载。这就是 streaming。

### 两种类型

【板书】
```
1. Real-time Streaming（实时流）：
   - 直播、视频会议
   - 数据被即时生成和传输
   - 对延迟要求极高

2. On-demand Streaming（点播流）：
   - B 站、Netflix、Spotify
   - 内容预先录制好，存储在服务器上
   - 用户随时播放
```

### Streaming vs Download

| | Streaming | Download |
|:---|:---|:---|
| 什么时候能用？ | 几乎立即 | 下载完才能用 |
| 占用本地存储？ | 很少（缓冲区） | 占用完整文件大小 |
| 网络中断？ | 播放卡顿/停止 | 不影响已下载的部分 |
| 需要持续网络？ | 是 | 下载后不需要 |

### Buffering（缓冲）

为了避免网速波动导致卡顿，播放器会提前下载一小段数据放在**缓冲区（Buffer）** 里。当你的网速暂时变慢，缓冲区里的数据还能维持一会儿播放，给网络恢复的时间。

## 本讲小结（约 2 分钟）

最后一讲的内容比较多，我们快速回顾：

1. **Internet ≠ WWW**——Internet 是基础设施，WWW 是运行其上的服务
