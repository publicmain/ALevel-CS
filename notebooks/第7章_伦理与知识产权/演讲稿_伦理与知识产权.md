## 开场：技术的"灰色地带"（约 5 分钟）

同学们好！今天我们要讲一个和前面几章完全不同的话题——不是技术实现，而是**技术背后的伦理问题**。

前面的章节告诉你们"技术能做什么"，今天这节课要讨论的是"技术应该做什么"和"技术不应该做什么"。

【提问】我先问大家一个问题：你在网上看到一首很好听的歌，你可以免费下载它。你会下载吗？为什么？

这个问题没有简单的"标准答案"。有人觉得"反正不会被发现"，有人觉得"这是盗版，不道德"。这就是伦理的特点——它不是法律那样黑白分明，很多时候是灰色地带。

【板书】
```
伦理 (Ethics) = 关于"什么是对的、什么是错的"的原则和标准
计算机伦理 (Computer Ethics) = 这些原则在信息技术领域的应用

注意区别：
- 法律 (Law)：由政府制定，违反会受到法律制裁
- 伦理 (Ethics)：由社会/行业制定的道德标准，违反可能不犯法但不道德
- 有些行为合法但不道德（比如收集用户数据卖给广告商）
- 有些行为违法但有人认为道德（比如公布政府的不当行为）
```

### 真实案例

让我们看一个著名案例：

【板书】
```
Facebook-Cambridge Analytica 事件 (2018)
1. 一个"性格测试" App 收集了用户及其好友的数据
2. 这些数据被卖给 Cambridge Analytica 公司
3. 该公司利用数据为政治竞选定向投放广告
4. 涉及约 8700 万用户
→ Facebook 被罚 50 亿美元
→ 引发全球对数据隐私的关注
```

这个事件中，Facebook 有没有直接"偷"用户数据？技术上说没有——用户自己同意了条款。但是，用户真的理解自己同意了什么吗？一个"性格测试"怎么变成了政治工具？这就是伦理问题——技术上合法，但道德上有争议。

## 第一节：专业伦理机构（约 5 分钟）

既然伦理问题这么复杂，谁来告诉 IT 从业者什么能做、什么不能做？这就是**专业伦理机构（Professional Ethical Bodies）**的作用。

考试中重点需要知道两个组织：

【板书】
```
BCS (British Computer Society) —— 英国计算机学会
IEEE (Institute of Electrical and Electronics Engineers) —— 电气电子工程师学会

它们的共同使命：
1. 制定行为准则 (Code of Conduct) —— 告诉从业者什么是正确的行为
2. 推动专业发展 —— 提供培训、认证
3. 维护公共利益 —— 确保技术服务于社会
```

BCS 的行为准则可以归纳为四大原则：

【板书】
```
BCS 行为准则四大原则：
1. Public Interest（公共利益优先）
   你的工作不能危害公众健康、安全或环境

2. Professional Competence（专业能力与诚信）
   只做你有能力做的事，不能假装自己会

3. Duty to Relevant Authority（对相关权威的义务）
   遵守法律和合同，对雇主和客户尽责

4. Duty to the Profession（对职业的义务）
   维护行业声誉，帮助同行提升
```

## 第二节：伦理场景讨论——电车难题的 IT 版（约 8 分钟）

现在我们来做一些伦理场景分析。这些场景没有标准答案，重要的是你的**思考过程**和**论证能力**。

### 场景一：自动驾驶的电车难题

【提问】一辆自动驾驶汽车正在高速行驶。前方突然出现一群行人。此时汽车有两个选择：
- A：直行，撞上行人（可能造成多人伤亡）
- B：急转弯，撞上路边的墙（乘客可能受伤或死亡）

如果你是编写这辆车 AI 算法的程序员，你怎么选？

这就是著名的"电车难题"（Trolley Problem）在自动驾驶领域的版本。这不仅仅是一个哲学问题——特斯拉、Waymo 等公司的工程师**真的需要**在代码里做出这种决策。

关键的伦理思考：
- 功利主义视角：选择伤亡最少的方案
- 义务论视角：不能把乘客当作牺牲品
- 谁来为 AI 的决策负责？程序员？公司？车主？

### 场景二：你发现了公司代码中的安全漏洞

【提问】你是一名程序员，发现公司的软件有一个严重的安全漏洞，可能导致百万用户数据泄露。你报告给了经理，但经理说"修复太贵了，先不管"。你怎么办？

- 服从老板？（对雇主尽责）
- 向监管机构举报？（公共利益优先）
- 辞职？（保护自己）

BCS 的准则怎么说？**公共利益优先**。当雇主的利益和公众的安全冲突时，你有义务保护公众。这叫做"吹哨（Whistleblowing）"。

## 第三节：版权法与软件许可证（约 10 分钟）

### 什么是版权？

【板书】
```
版权 (Copyright)：
- 自动授予原创作品创作者的法律保护
- 不需要注册——你写的代码从创作那一刻起就受版权保护
- 保护"表达形式"而非"想法"
  （你不能版权保护"排序"这个想法，但可以版权保护你写的排序代码）

版权保护的范围：
- 源代码和目标代码
- 文档和用户手册
- 网站设计和内容
- 数据库结构
- 音乐、图片、视频、文字

版权不保护的：
- 算法思想（但可以申请专利）
- 数据本身
- 编程语言本身
```

### 软件许可证类型

当你拥有代码的版权后，你可以通过**许可证（License）**来决定别人能怎么使用你的软件。

【板书】
```
四种主要的许可证类型：

1. 商业许可 (Commercial / Proprietary License)
   - 付费购买使用权
   - 不能查看源码、不能修改、不能再分发
   - 例子：Microsoft Office, Adobe Photoshop

2. 开源许可 (Open Source License)
   - 源码公开，任何人可以查看、修改、再分发
   - 可能有附加条件（如 GPL 要求修改后也必须开源）
   - 例子：Linux, Firefox, Python

3. 免费软件 (Freeware)
   - 免费使用，但通常不能看到源码
   - 作者保留版权
   - 例子：Skype, Adobe Reader

4. 共享软件 (Shareware)
   - 先免费试用，满意后付费
   - 试用版可能有功能限制或时间限制
   - 例子：WinRAR（理论上要付费，但...）
```

### 开源 vs 商业：哪个更好？

【提问】你们觉得开源软件和商业软件各有什么优缺点？

【板书】
```
开源软件的优势：
+ 免费（降低成本）
+ 透明（任何人可以审查代码找安全漏洞）
+ 灵活（可以根据需要修改）
+ 社区支持（全球开发者贡献）

开源软件的劣势：
- 可能缺乏专业技术支持
- 质量参差不齐
- 文档可能不完善
- 界面可能不如商业软件友好

商业软件则恰好相反。
```

## 第四节：人工智能——机遇与挑战（约 12 分钟）

最后一部分，我们来聊聊近年来最热门的话题——**人工智能（AI）**。

### AI 是什么？

【板书】
```
人工智能 (Artificial Intelligence, AI)
= 让计算机模拟人类智能行为的技术
= 包括学习、推理、感知、决策等能力

AI 的主要分支：
- 机器学习 (Machine Learning)：从数据中学习规律
- 深度学习 (Deep Learning)：模拟神经网络的多层学习
- 自然语言处理 (NLP)：理解和生成人类语言
- 计算机视觉 (Computer Vision)：理解图像和视频
- 机器人技术 (Robotics)：让机器在物理世界中行动
```

### AI 的社会影响

【提问】你们每天在用 AI 吗？手机的面部识别、推荐算法、语音助手——这些都是 AI。那 AI 对社会有哪些影响呢？

【板书】
```
AI 的积极影响：
+ 医疗：AI 辅助诊断，比如分析 X 光片发现早期癌症
+ 教育：个性化学习路径，智能辅导
+ 交通：自动驾驶减少交通事故（人为因素导致 90%+ 事故）
+ 效率：自动化重复性工作，提高生产力
+ 科研：加速药物研发、气候模型计算

AI 的负面影响/风险：
- 就业：自动化可能取代大量工作岗位
- 偏见：AI 从有偏见的数据中学习，可能放大歧视
  （例如招聘 AI 歧视女性，因为训练数据中男性员工占多数）
- 隐私：面部识别技术可能被滥用于监控
- 安全：AI 生成的深度伪造 (Deepfake) 可以制造虚假视频
- 环境：训练大型 AI 模型消耗巨量能源

伦理AI原则：
- 公平 (Fairness)：AI 不应歧视任何群体
- 透明 (Transparency)：AI 的决策过程应该可以解释
- 隐私 (Privacy)：AI 应该尊重个人数据
- 安全 (Safety)：AI 不应对人造成伤害
- 问责 (Accountability)：AI 出了问题要有人负责
```

### AI 的经济影响

【提问】你们觉得 AI 会让哪些工作消失？会创造哪些新工作？

历史告诉我们：每一次技术革命都会消灭一些旧工作、创造一些新工作。汽车消灭了马车夫，但创造了出租车司机、汽车修理工、加油站员工。

AI 也是一样。可能消失的工作：简单数据录入、基础客服、部分翻译工作。可能出现的新工作：AI 训练师、AI 伦理审查员、数据标注工程师。

关键不是"AI 会不会取代人"，而是"什么样的人不会被 AI 取代"——答案是：那些能够与 AI 协作、发挥人类独特优势（创造力、同理心、复杂判断）的人。
