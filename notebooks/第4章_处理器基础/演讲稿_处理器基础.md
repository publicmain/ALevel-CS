# 第一讲：冯诺依曼架构与 CPU

**（对应 Notebook：01_冯诺依曼架构与CPU.ipynb）**

---

## 开场（约 3 分钟）

同学们好！今天我们要深入到计算机最核心的部件——CPU。

我先讲一个小故事。在 1940 年代，世界上最早的几台计算机——比如 ENIAC——有一个巨大的问题。

【提问】你们猜，ENIAC 要换一个计算任务，需要做什么？

答案是：**重新接线**。对，你没听错。ENIAC 的"程序"是通过成千上万根电缆的物理连接来实现的。你想让它算一个新的公式？好的，请几个工程师花几天时间重新插线。

这就是所谓的"硬连线程序（Hardwired Program）"——程序被固化在硬件中，修改程序就要修改硬件。

然后，一个叫**冯诺依曼（John von Neumann）** 的数学家提出了一个改变世界的想法……

---

## 第一节：冯诺依曼的革命——存储程序概念（约 8 分钟）

### 核心思想

【板书】
```
存储程序概念（Stored Program Concept）：

1. 程序（指令）和数据都以二进制形式存储在同一个内存中
2. 指令按顺序从内存中取出并执行
3. 要换程序？只需要在内存中加载新程序——不需要改硬件！
```

这听起来理所当然？但在当时是革命性的。从此，计算机不再是"一次性工具"，而是"通用机器"——同一台计算机，通过加载不同的程序，可以干任何事情。

### 冯诺依曼架构的五大组件

【板书】
```
冯诺依曼架构的核心组件：

┌──────────────────────────────────┐
│             CPU                   │
│  ┌────────┐  ┌────────┐         │
│  │  CU    │  │  ALU   │         │
│  │控制单元│  │算术逻辑│         │
│  └────────┘  └────────┘         │
│       ┌────────────┐            │
│       │  Registers │            │
│       │   寄存器   │            │
│       └────────────┘            │
│            Clock                │
└──────────────────────────────────┘
         ↕ 总线系统（Bus）
┌──────────────────────────────────┐
│     Main Memory (IAS)            │
│     存储程序和数据                │
└──────────────────────────────────┘
         ↕ 总线系统（Bus）
┌──────────────────────────────────┐
│     Input/Output Devices         │
└──────────────────────────────────┘
```

### 冯诺依曼 vs 哈佛架构

【板书】

| 特征 | 冯诺依曼 | 哈佛 |
|:---|:---|:---|
| 内存 | 程序和数据共享同一内存 | 程序和数据有独立内存 |
| 总线 | 共享总线 | 独立总线 |
| 优点 | 设计简单，灵活 | 可同时取指令和取数据，更快 |
| 缺点 | 冯诺依曼瓶颈 | 设计复杂，成本高 |

**冯诺依曼瓶颈（Von Neumann Bottleneck）**：因为指令和数据共用同一条总线，CPU 不能同时取指令和取数据——必须轮流来。这就像一条单车道公路，两辆车不能同时通过。

现代计算机通常混合使用两种架构——内部缓存使用哈佛架构（L1 缓存分为指令缓存和数据缓存），外部主内存使用冯诺依曼架构。

## 第二节：CPU 核心组件详解（约 12 分钟）

### 2.1 ALU（算术逻辑单元）

【板书】
```
ALU (Arithmetic & Logic Unit)：
- CPU 的"计算引擎"
- 负责所有的数学运算：加减乘除
- 负责所有的逻辑运算：AND、OR、NOT、比较
- 它不做决策——只是执行 CU 告诉它的操作
```

### 2.2 CU（控制单元）

【板书】
```
CU (Control Unit)：
- CPU 的"指挥官"
- 负责：
  1. 解码指令（把二进制指令翻译成具体操作）
  2. 协调各部件（告诉 ALU 该做什么运算，告诉内存该读还是写）
  3. 产生控制信号（通过控制总线发送给各部件）
  4. 管理取指-执行周期（下一讲的核心内容）
```

### 2.3 时钟（System Clock）

【板书】
```
System Clock：
- 产生规律的电脉冲信号，同步 CPU 所有操作
- 每个脉冲 = 一个时钟周期
- 时钟频率 = 每秒多少个周期（单位：Hz）
- 例如 3.5 GHz = 每秒 35 亿个时钟周期
```

时钟就像一个乐队指挥手中的节拍器——所有乐器（CPU 部件）必须跟着同一个节拍工作，不然就乱了。

### 2.4 寄存器（Registers）

寄存器是 CPU 内部**最快的存储单元**——比 RAM 快几十倍。但数量很少，容量很小。

#### 通用寄存器

可以临时存放正在处理的数据，程序员可以直接使用。

#### 专用寄存器——考试重点！

【板书】
```
考试必考的专用寄存器：

PC (Program Counter)
  → 存储下一条要执行的指令的内存地址
  → 每取一条指令后自动 +1

CIR (Current Instruction Register)
  → 存储当前正在执行的指令
  → 指令在这里被解码

MAR (Memory Address Register)
  → 存储要访问的内存地址
  → CPU 想读/写内存时，先把地址放到 MAR

MDR (Memory Data Register) / MBR (Memory Buffer Register)
  → 存储从内存读出的数据，或即将写入内存的数据
  → 是 CPU 和内存之间的"数据传送窗口"

ACC (Accumulator)
  → 存储 ALU 运算的结果
  → 也叫"累加器"

IX (Index Register)
  → 用于变址寻址（Indexed Addressing）
  → 存储偏移量
```

【提问】PC、MAR、MDR、CIR、ACC——这 5 个寄存器你们一定要记住。我教你们一个记忆方法：

想象 CPU 处理一条指令的过程就像去图书馆借书：
- **PC** = 你的借书清单上的下一本书的编号
- **MAR** = 你告诉图书管理员："我要找书架号 XXX 的书"
- **MDR** = 管理员从书架上拿下来递给你的那本书
- **CIR** = 你正在阅读的那本书
- **ACC** = 你读完后做的笔记

## 第三节：总线系统（约 8 分钟）

### 什么是总线？

**总线（Bus）** = 一组并行的导线，用于在 CPU、内存、I/O 设备之间传输数据。

### 三种总线

【板书】
```
1. 地址总线 (Address Bus) — 单向（CPU → 内存/IO）
   - CPU 用它告诉内存："我要访问哪个地址"
   - 宽度决定可寻址空间
   - n 位地址总线 → 可寻址 2^n 个内存单元

2. 数据总线 (Data Bus) — 双向
   - CPU 和内存/IO 之间传输实际数据
   - 宽度决定一次能传多少数据
   - n 位数据总线 → 一次传输 n 位数据

3. 控制总线 (Control Bus) — 双向
   - 传输控制信号：读/写、时钟、中断请求等
   - 协调各部件的工作
```

【演示】运行 Notebook 01 中的 CPU 架构可视化图。

大家看这张图。CPU 在上方，内存在左下，I/O 在右下。三条颜色不同的总线连接它们。注意地址总线的箭头是单向的（从 CPU 出发），数据总线和控制总线是双向的。

### 总线宽度与性能

【板书】
```
地址总线宽度 = n 位
→ 可寻址内存 = 2^n 个地址

例题：
32 位地址总线 → 2^32 = 4,294,967,296 个地址
如果每个地址对应 1 Byte → 最大内存 = 4 GB

这就是为什么 32 位操作系统最多只能用 4GB 内存！
64 位地址总线 → 2^64 ≈ 16 EB（理论上）
```

【演示】运行 Notebook 01 中的总线宽度与可寻址空间计算代码。

【提问】如果地址总线从 32 位加宽到 33 位，可寻址内存变成多少？

答案是翻倍！2^33 = 8 GB。每多一位，可寻址空间翻一倍——这就是二进制的指数增长。

## 第四节：影响 CPU 性能的因素（约 5 分钟）

【板书】
```
影响 CPU 性能的 4 个因素：

1. 时钟频率 (Clock Speed)
   - 越高 → 每秒执行的周期越多 → 越快
   - 但不能无限提高（散热问题）

2. 核心数 (Number of Cores)
   - 多核 CPU = 一个 CPU 里有多个独立的处理单元
   - 理论上 4 核应该比单核快 4 倍
   - 实际上受限于程序能否并行化

3. 缓存 (Cache)
   - CPU 内部的高速存储（SRAM）
   - 层级：L1（最快最小）→ L2 → L3（较慢较大）
   - 缓存命中率越高，CPU 等待内存的时间越少

4. 总线宽度 (Bus Width)
   - 数据总线越宽 → 一次传的数据越多
   - 地址总线越宽 → 能访问的内存越大
```

# 第二讲：取指执行周期

**（对应 Notebook：02_取指执行周期.ipynb）**

---

## 开场（约 3 分钟）

同学们好！上节课我们认识了 CPU 的各个组件和寄存器。今天我们要回答一个更根本的问题：**CPU 是怎么一步一步执行程序的？**

你可以把 CPU 想象成一个超级快的"厨师"。它面前有一本菜谱（程序），菜谱上一条一条写着步骤（指令）。厨师每次做三件事：
1. **看一条**菜谱（取指令）
2. **理解**这条步骤要求干什么（解码）
3. **动手做**（执行）

然后翻到下一条，重复。这就是**取指-执行周期（Fetch-Execute Cycle，也叫 FDE Cycle）**——CPU 永远在这三个阶段中循环。

---

## 第一节：三个阶段概览（约 3 分钟）

【板书】
```
取指-执行周期的三个阶段：

1. Fetch（取指）：从内存中取出下一条指令
2. Decode（解码）：CU 解读指令，确定要做什么操作
3. Execute（执行）：执行指令（可能是算术运算、数据传输等）

→ 然后回到 Fetch，如此循环，直到程序结束或遇到 HALT
```

---

## 第二节：Fetch 阶段详解（约 10 分钟）

这是考试的绝对重点。你必须能一步一步描述 Fetch 阶段的每个步骤，包括涉及哪些寄存器。

【板书】
```
Fetch 阶段 — 4 个步骤：

Step 1：MAR ← [PC]
  PC 中的地址被复制到 MAR
  （"告诉内存：我要读这个地址的内容"）

Step 2：PC ← [PC] + 1
  PC 自动加 1，指向下一条指令的地址
  （"书签翻到下一页"）

Step 3：MDR ← [[MAR]]
  内存根据 MAR 中的地址，把对应的数据送到 MDR
  （"内存把指令递给 CPU"）
  同时，控制总线发出 READ 信号

Step 4：CIR ← [MDR]
  MDR 中的指令被复制到 CIR
  （"把指令放到'当前正在处理'的位置"）
```

注意几个细节：
- Step 2 为什么是在取指令的同时就把 PC +1？因为效率！不需要等执行完才更新 PC。
- 方括号 `[PC]` 表示"PC 寄存器中的值"，双方括号 `[[MAR]]` 表示"MAR 中的值所指向的内存地址中的值"。

【提问】如果 PC 现在的值是 100，那 Fetch 阶段结束后：
- MAR 中存的是什么？100
- PC 中存的是什么？101
- MDR 中存的是什么？内存地址 100 中的指令
- CIR 中存的是什么？同样是那条指令

---

## 第三节：Decode 和 Execute 阶段（约 5 分钟）

### Decode 阶段

【板书】
```
Decode 阶段：
- CU 读取 CIR 中的指令
- 指令分为两部分：
  操作码 (Opcode) | 操作数 (Operand)
  "做什么"        | "对什么做"
- CU 解码操作码，确定需要哪些部件参与
```

### Execute 阶段

执行阶段取决于指令类型：

【板书】
```
Execute 阶段（不同指令类型）：

- 数据加载（LDD）：从内存读数据到 ACC
  MAR ← 操作数中的地址
  MDR ← [[MAR]]
  ACC ← [MDR]

- 算术运算（ADD）：ALU 执行运算
  MAR ← 操作数中的地址
  MDR ← [[MAR]]
  ACC ← [ACC] + [MDR]

- 数据存储（STO）：把 ACC 的值写入内存
  MAR ← 操作数中的地址
  MDR ← [ACC]
  [[MAR]] ← [MDR]

- 跳转（JMP）：修改 PC
  PC ← 操作数中的地址
```

---

## 第四节：寄存器传输记法 RTN（约 8 分钟）

### 符号说明

【板书】
```
RTN (Register Transfer Notation) 符号：
- ← 表示"赋值"（左边 ← 右边的值）
- [X] 表示"X 中存储的值"
- [[X]] 表示"X 中的值作为地址，该地址中存储的值"
```

### 完整的 Fetch 阶段 RTN

【板书】
```
MAR ← [PC]
PC  ← [PC] + 1
MDR ← [[MAR]]
CIR ← [MDR]
```

考试时经常会让你用 RTN 描述取指阶段。这四行你必须背得滚瓜烂熟。

### Execute 阶段 RTN 示例

【板书】
```
指令：LDD 200（加载地址 200 的值到 ACC）
  MAR ← CIR 的操作数部分 (即 200)
  MDR ← [[MAR]]     即 MDR ← [200]
  ACC ← [MDR]

指令：ADD 201（把地址 201 的值加到 ACC）
  MAR ← 201
  MDR ← [[MAR]]     即 MDR ← [201]
  ACC ← [ACC] + [MDR]

指令：STO 300（把 ACC 的值存到地址 300）
  MAR ← 300
  MDR ← [ACC]
  [[MAR]] ← [MDR]   即 [300] ← [MDR]
```

## 第五节：中断（约 10 分钟）

### 什么是中断？

CPU 正在忙着执行程序，突然键盘收到一个按键、或者打印机说"我的纸用完了"——这些事件需要 CPU 紧急处理。这就是**中断（Interrupt）**。

【板书】
```
中断 (Interrupt)：
- 来自硬件或软件的信号，要求 CPU 暂停当前工作，去处理紧急事务
- 中断检查发生在每个 FDE 周期的 Execute 阶段之后
```

### 为什么需要中断？

如果没有中断机制，CPU 就必须不断**轮询（Polling）** 每个设备："键盘你有输入吗？""打印机你好了吗？"——这极其浪费时间。

有了中断，设备可以主动通知 CPU："我这里有事！"CPU 只在被通知时才去处理——就像手机响了你才接电话，而不是每隔 5 秒拿起来看一下。

### 中断处理流程

【板书】
```
中断处理流程：
1. CPU 完成当前指令的执行
2. 检查是否有中断信号
3. 如果有：
   a. 保存当前状态（PC、ACC、寄存器等）到栈（Stack）
   b. 加载中断服务程序（ISR）的地址到 PC
   c. 执行 ISR
   d. ISR 执行完毕后，从栈中恢复之前的状态
   e. 继续执行被中断的程序
4. 如果没有中断：继续下一个 FDE 周期
```

### 中断的类型

【板书】
```
中断类型：
- 硬件中断：键盘按键、鼠标点击、打印机就绪
- 软件中断：程序主动请求操作系统服务（系统调用）
- 定时器中断：时钟芯片定期发出，用于进程调度
- 错误中断：除以零、非法内存访问
```

### 中断优先级

不是所有中断都一样紧急。电源故障显然比键盘按键更紧急。所以中断有优先级——高优先级可以打断低优先级。

## 第六节：完整的 FDE 周期（含中断）（约 3 分钟）

【板书】
```
完整的取指-执行周期：

START:
  [Fetch]
    MAR ← [PC]
    PC  ← [PC] + 1
    MDR ← [[MAR]]
    CIR ← [MDR]
  [Decode]
    CU 解码 CIR 中的指令
  [Execute]
    执行指令
  [Interrupt Check]
    如果有中断 → 保存状态 → 跳转到 ISR
    如果没有 → 回到 Fetch
  GOTO START
```

# 第三讲：汇编语言

**（对应 Notebook：03_汇编语言.ipynb）**

---

## 开场（约 3 分钟）

同学们好！前两节课我们知道了 CPU 是怎么一步步执行指令的。但那些指令到底长什么样？

CPU 能直接理解的只有**机器码（Machine Code）**——纯粹的二进制数字，比如 `0001 0000 1100 1000`。

【提问】你觉得让程序员直接写这样的二进制代码，方便吗？

当然不方便！谁能记住 0001 代表"加载"、0010 代表"加法"？所以人们发明了**汇编语言（Assembly Language）**——用助记符（mnemonics）代替二进制操作码。

比如把 `0001` 替换成 `LDD`（Load Direct），把 `0010` 替换成 `ADD`。含义一样，但人类更容易读写。

---

## 第一节：机器码 vs 汇编语言（约 5 分钟）

【板书】
```
机器码 (Machine Code)：
  0001 0000 11001000     ← CPU 直接执行
  0010 0000 11001001
  0100 0000 11001010

汇编语言 (Assembly Language)：
  LDD 200               ← 人能读懂的助记符
  ADD 201
  STO 202

高级语言 (High-level Language)：
  result = a + b         ← 最接近人类语言

转换关系：
  汇编语言 —[汇编器 Assembler]→ 机器码
  高级语言 —[编译器/解释器]→ 机器码
```

汇编语言和机器码是**一一对应**的——每一条汇编指令恰好对应一条机器码指令。这和高级语言不同——一行 Python 可能对应几十条机器码。

---

## 第二节：指令分类（约 8 分钟）

Cambridge 考试纲要定义了一套标准的汇编指令集，你们必须记住。

### 2.1 数据传输指令

【板书】
```
LDM #n    ← 立即寻址：把数值 n 直接加载到 ACC
LDD addr  ← 直接寻址：把内存地址 addr 中的值加载到 ACC
LDI addr  ← 间接寻址：addr 中存的是另一个地址，加载那个地址的值
LDX addr  ← 变址寻址：加载 (addr + IX) 处的值
LDR #n    ← 把数值 n 加载到 IX（Index Register）
STO addr  ← 把 ACC 中的值存储到内存地址 addr
MOV reg   ← 把 ACC 中的值移动到指定寄存器
```

### 2.2 算术指令

【板书】
```
ADD addr  ← ACC ← ACC + [addr]（直接寻址）
ADD #n    ← ACC ← ACC + n（立即寻址）
SUB addr  ← ACC ← ACC - [addr]
INC reg   ← 寄存器的值 +1
DEC reg   ← 寄存器的值 -1
```

### 2.3 比较与跳转

【板书】
```
CMP addr  ← 比较 ACC 与 [addr] 的值（设置标志位）
CMP #n    ← 比较 ACC 与 n
CMI addr  ← 间接寻址比较
JMP addr  ← 无条件跳转：PC ← addr
JPE addr  ← 如果比较结果相等则跳转
JPN addr  ← 如果比较结果不等则跳转
```

### 2.4 输入输出与结束

【板书】
```
IN        ← 从键盘读入一个值到 ACC
OUT       ← 输出 ACC 的值到屏幕
END       ← 程序结束（相当于 HALT）
```

## 第三节：五种寻址方式——关键考点！（约 12 分钟）

这是本讲最重要的内容。同一个操作（比如"加载"），根据操作数的含义不同，有 5 种寻址方式。

### 3.1 立即寻址（Immediate Addressing）

【板书】
```
LDM #5
含义：把数值 5 直接放入 ACC
操作数就是数据本身

特点：
- 最快（不需要访问内存）
- 操作数前面有 # 标记
- 限制：只能处理小数值（受操作数位数限制）
```

类比：老师直接告诉你答案："答案是 5。"

### 3.2 直接寻址（Direct Addressing）

【板书】
```
LDD 200
含义：去内存地址 200，取出里面的值，放入 ACC
操作数是内存地址

特点：
- 需要访问一次内存
- 最常用的寻址方式
```

类比：老师告诉你："答案在第 200 页。"你需要翻到第 200 页才能看到答案。

### 3.3 间接寻址（Indirect Addressing）

【板书】
```
LDI 200
含义：
  先去地址 200，取出里面的值（假设是 500）
  再去地址 500，取出那里的值，放入 ACC
操作数是一个"指针"——它存的不是数据，而是数据的地址

特点：
- 需要访问两次内存（更慢）
- 非常灵活——可以间接引用任何地址
```

类比：老师告诉你："第 200 页写着答案在哪里。"你翻到第 200 页，看到写着"去第 500 页"。然后你再翻到第 500 页才看到答案。

### 3.4 变址寻址（Indexed Addressing）

【板书】
```
LDX 200
含义：
  实际访问的地址 = 200 + IX（Index Register 中的值）
  如果 IX = 3，则访问地址 203

特点：
- 非常适合遍历数组
- 基地址 + 偏移量（IX）
- 通过改变 IX 的值，可以访问数组的不同元素
```

类比：老师说："从第 200 页开始数，往后数 IX 页。"如果 IX=0，读第 200 页；IX=1，读第 201 页；IX=2，读第 202 页。这就是遍历！

### 3.5 相对寻址（Relative Addressing）

【板书】
```
JMP #offset  （相对跳转）
含义：
  新的 PC = 当前 PC + offset
  offset 可以是正数（向前跳）或负数（向后跳）

特点：
- 代码可以被加载到内存的任何位置运行（位置无关代码）
- 常用于跳转指令
```

类比：不是"跳到第 200 页"，而是"从当前页往后翻 5 页"。不管你当前在哪一页，都能正确跳转。

### 5 种寻址方式对比

【演示】运行 Notebook 03 中的寻址方式可视化图。

【板书】

| 寻址方式 | 操作数含义 | 内存访问次数 | 用途 |
|:---|:---|:---|:---|
| 立即 | 数据本身 | 0 | 常量 |
| 直接 | 数据的地址 | 1 | 一般变量 |
| 间接 | 数据地址的地址 | 2 | 指针、动态引用 |
| 变址 | 基地址 + IX | 1 | 数组遍历 |
| 相对 | 偏移量 | 0 | 跳转、循环 |

## 第四节：两遍汇编器（约 8 分钟）

### 问题：前向引用

【板书】
```
问题代码：
  Line 1:  JMP LOOP    ← LOOP 标签还没有定义！
  Line 2:  ADD 100
  Line 3:  LOOP: SUB 101  ← LOOP 在这里才定义

汇编器在第 1 行遇到 LOOP 时，还不知道 LOOP 对应的地址是什么。
这就是"前向引用"问题。
```

解决方案：**两遍扫描（Two-Pass Assembler）**。

### 第一遍（Pass 1）：构建符号表

【板书】
```
Pass 1：
- 逐行扫描源代码
- 遇到标签（如 LOOP:）就记录：标签名 → 对应的内存地址
- 构建出一张"符号表（Symbol Table）"

符号表示例：
  LOOP → 地址 102
  END  → 地址 108
  DATA → 地址 200
```

### 第二遍（Pass 2）：生成机器码

【板书】
```
Pass 2：
- 再次逐行扫描源代码
- 把每条指令翻译成机器码
- 遇到标签引用时，查符号表替换成实际地址
- 输出最终的机器码
```

【演示】运行 Notebook 03 中的两遍汇编器模拟器。

大家看模拟器的输出。Pass 1 结束后，我们得到了一张符号表。Pass 2 利用这张符号表，把所有标签替换成了实际地址，生成了纯数字的机器码。

## 第五节：汇编程序追踪（约 5 分钟）

考试经常会给你一段汇编代码，让你用 **Trace Table** 追踪程序执行过程。

【板书】
```
Trace Table 的列通常包括：
PC | CIR (指令) | ACC | IX | 其他寄存器 | 输出 | 相关内存地址的值

每一行代表执行一条指令后各个值的变化。
```

【演示】运行 Notebook 03 中的 Trace Table 工具，追踪一个从 1 数到 5 的小程序。

关键技巧：
- 严格按照一条指令一行
- 只更新本条指令改变的值，其他值保持不变
- CMP 指令不改变任何寄存器的值——只设置标志位

## 开场（约 3 分钟）

同学们好！今天是处理器基础这一章的最后一讲。我们要学习一项非常实用的底层技能——**位操作（Bit Manipulation）**。

【提问】你们知道为什么游戏开发者和嵌入式系统工程师特别喜欢位操作吗？

因为位操作是**最快的运算**——它直接在二进制位上操作，比加减乘除都快。而且在控制硬件设备时，你经常需要精确地设置或检查某一个特定的位——这就是位操作的用武之地。

今天的内容分两大块：**移位运算**和**位掩码**。

---

## 第一节：二进制移位（约 15 分钟）

### 什么是移位？

移位就是把二进制数的所有位统一向左或向右移动若干位。

### 1.1 逻辑移位（Logical Shift）

#### 逻辑左移

【板书】
```
逻辑左移（Logical Left Shift）：
- 所有位向左移 n 位
- 右边空出来的位补 0
- 左边移出去的位丢弃

例子：00101100 (44) 左移 1 位
  0 ← 01011000
     = 88

效果：相当于 × 2^n
  左移 1 位 = ×2
  左移 2 位 = ×4
  左移 3 位 = ×8
```

#### 逻辑右移

【板书】
```
逻辑右移（Logical Right Shift）：
- 所有位向右移 n 位
- 左边空出来的位补 0
- 右边移出去的位丢弃

例子：00101100 (44) 右移 1 位
  00010110 → = 22

效果：相当于整数除以 2^n（向下取整）
  右移 1 位 = ÷2
  右移 2 位 = ÷4
```

【演示】运行 Notebook 04 中的逻辑移位可视化代码。

【提问】为什么左移一位等于乘 2？

想想十进制：123 左移一位变成 1230——乘了 10。二进制中，左移一位就是乘 2（二进制的"基数"是 2）。

### 1.2 算术移位（Arithmetic Shift）

逻辑移位有一个问题——它不能正确处理**有符号数（补码表示）**。

【板书】
```
算术右移（Arithmetic Right Shift）：
- 与逻辑右移的区别：左边补的不是 0，而是符号位（最高位）
- 如果是正数（最高位=0）→ 补 0（和逻辑右移相同）
- 如果是负数（最高位=1）→ 补 1（保持负号）

例子：11010000 (-48 的补码) 算术右移 1 位
  11101000 = -24  ✓ 正确的 ÷2

如果用逻辑右移：
  01101000 = 104  ✗ 错误！符号都变了
```

算术左移和逻辑左移在大多数情况下是相同的——左移不涉及符号位的处理问题。

### 1.3 循环移位（Cyclic Shift）

【板书】
```
循环移位（Cyclic Shift / Rotate）：
- 移出去的位不丢弃——从另一端重新进来
- 循环左移：左边移出的位从右边进来
- 循环右移：右边移出的位从左边进来

例子：10110001 循环左移 1 位
  → 01100011（最左边的 1 跑到了最右边）

特点：不会丢失任何位——信息完全保留
```

【演示】运行 Notebook 04 中的算术移位和循环移位可视化。

### 移位总结

【板书】

| 类型 | 空位填充 | 移出的位 | 适用场景 |
|:---|:---|:---|:---|
| 逻辑移位 | 补 0 | 丢弃 | 无符号数的乘除 |
| 算术移位 | 补符号位 | 丢弃 | 有符号数的乘除 |
| 循环移位 | 用移出的位 | 循环回来 | 加密、位检查 |

## 第二节：位掩码（约 12 分钟）

位掩码（Bit Masking）= 用一个特定的二进制模式（mask）和逻辑运算来操作特定的位。

### 2.1 AND 掩码——测试/清除位

【板书】
```
AND 掩码规则：
- 1 AND x = x（保留原值）
- 0 AND x = 0（清零）

用途 1：测试某一位是 0 还是 1
  数据：    1 0 1 1 0 1 1 0
  掩码：    0 0 0 0 1 0 0 0  ← 只在第 3 位是 1
  AND 结果：0 0 0 0 0 0 0 0  → 第 3 位是 0

  数据：    1 0 1 1 1 1 1 0
  掩码：    0 0 0 0 1 0 0 0
  AND 结果：0 0 0 0 1 0 0 0  → 第 3 位是 1

用途 2：清除特定位（强制置 0）
  想把第 5 位清零：用 AND 掩码 11011111
```

### 2.2 OR 掩码——设置位

【板书】
```
OR 掩码规则：
- 0 OR x = x（保留原值）
- 1 OR x = 1（强制置 1）

用途：把特定位设置为 1
  数据：    1 0 1 0 0 1 1 0
  掩码：    0 0 0 0 1 0 0 0  ← 想把第 3 位设为 1
  OR 结果： 1 0 1 0 1 1 1 0  ← 第 3 位变成了 1，其他不变
```

### 2.3 XOR 掩码——翻转位

【板书】
```
XOR 掩码规则：
- 0 XOR x = x（保留原值）
- 1 XOR x = NOT x（翻转）

用途：把特定位取反（0→1，1→0）
  数据：    1 0 1 1 0 1 1 0
  掩码：    0 0 0 0 1 1 0 0  ← 翻转第 2、3 位
  XOR 结果：1 0 1 1 1 0 1 0  ← 第 2、3 位翻转了

另一个神奇特性：XOR 两次恢复原值
  A XOR B XOR B = A
  这就是为什么 XOR 被广泛用于加密！
```

### 掩码总结

【板书】

| 操作 | 掩码中的 1 的效果 | 掩码中的 0 的效果 | 用途 |
|:---|:---|:---|:---|
| AND | 保留原值 | 清零 | 测试位、清除位 |
| OR | 强制置 1 | 保留原值 | 设置位 |
| XOR | 翻转 | 保留原值 | 翻转位、简单加密 |

这个表格一定要记住！考试时很多题目就是围绕这三个规则出的。

## 第三节：实际应用——设备控制（约 8 分钟）

### LED 控制寄存器

在嵌入式系统中，硬件设备通常通过一个**控制寄存器**来控制。每一位对应一个功能。

【板书】
```
例子：一个 8 位 LED 控制寄存器
位 7: LED 7
位 6: LED 6
...
位 0: LED 0

该位 = 1 → 对应的 LED 亮
该位 = 0 → 对应的 LED 灭

当前状态：10110010（LED 7, 5, 4, 1 亮）
```

【提问】如何在不影响其他 LED 的情况下，打开 LED 3？

用 OR 掩码！

```
当前状态：1 0 1 1 0 0 1 0
OR 掩码： 0 0 0 0 1 0 0 0
结果：    1 0 1 1 1 0 1 0  ← LED 3 打开了，其他不变
```

【提问】如何在不影响其他 LED 的情况下，关闭 LED 5？

用 AND 掩码！

```
当前状态：1 0 1 1 1 0 1 0
AND 掩码：1 1 0 1 1 1 1 1  ← 第 5 位是 0，其他都是 1
结果：    1 0 0 1 1 0 1 0  ← LED 5 关闭了，其他不变
```

【提问】如何检测 LED 2 是否亮着？

用 AND 掩码测试！

```
当前状态：1 0 0 1 1 0 1 0
AND 掩码：0 0 0 0 0 1 0 0  ← 只有第 2 位是 1
结果：    0 0 0 0 0 0 0 0  → 结果为 0，说明 LED 2 是灭的
```

## 第四节：移位与掩码的组合应用（约 3 分钟）

移位和掩码经常组合使用。比如：

【板书】
```
任务：读取一个 8 位寄存器的第 4-7 位（高 4 位）作为一个 4 位数值

步骤：
1. 先用 AND 掩码清除低 4 位：AND 11110000
2. 再右移 4 位：得到高 4 位的值

例子：寄存器值 10110101
  AND 11110000 = 10110000
  右移 4 位 = 00001011 = 11（十进制）
```

这种技巧在嵌入式编程中无处不在——一个寄存器的不同位段可能控制不同的功能，你需要用掩码和移位来"提取"你需要的那一段。
